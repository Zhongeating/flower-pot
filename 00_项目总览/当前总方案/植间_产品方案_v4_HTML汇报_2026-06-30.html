价值感受

可见变化新叶、花苞、开花带来时间感

Alternative Gap

现有桌面替代物只解决了一部分需求

桌面上并不缺“好看”或“可爱”的东西，缺的是能长期低负担存在、长期能被用户看见的东西。

替代物已解决没解决
普通绿植真实、便宜、容易买弱光下成长有限，变化不明显
假花/摆件好看、稳定、低维护没有真实生命和成长变化
电子桌宠/屏保反馈强、可爱、有陪伴感交互智能有限
鱼缸/微景观生命感强、观赏性强维护复杂、风险高、桌面适配难
情绪智能花盆可爱、互动、传播强保活、传感、续航不足，成长有限，智能有限
室内种植机保活强、种植稳定更像专业设备，不够个人桌面化
Opportunity Combination

机会在三个价值同时成立

真实 + 成长 + 低精力

价值用户为什么在意
真实它是真的活着，不是模拟内容
成长新叶、花苞、开花带来时间感和成就感
低精力不增加新的高频待办

植间要验证的组合假设

目标用户是否真的需要，并愿意为一盆能长期留在固定桌面、不用天天判断养护、还能明显成长开花的植物付费。

机会组合，不默认植间方案已成立。

Scene Conflict

想要桌面开花植物，却被困难阻挡

为什么是开花/成长型植物为什么桌面难成立
真实生命比桌面摆件更有关系感弱光导致营养不足，不开花、徒长或衰弱
花苞和开花比普通绿叶更有变化用户不想/不会判断浇水、光照、肥料
成长变化能形成持续关注土表潮湿、虫害、水渍会影响桌面
照料结果带来成就感维护动作会消耗本就不足的精力

低精力系统减轻用户投入，开花植物验证回报“可见成长”是否足以支撑用户付费。

User Jobs

保活是底座，成长和安宁是两条上层价值路径

GROWTH PATH

看到植物真的在成长

新叶、花苞、开花、照片时间线，让用户感到“我养的东西在变好”。

CALM PATH

获得低注意力的情绪安宁

植物有存在感，但不需要用户持续关注、频繁操作或被设备召唤。

共同底座：桌上有真植物，但不再养死、不增加高频待办保活解决体验中断，成长和安宁决定用户是否长期留下。
Early Users

面对养护难，用户不同策略

策略可能含义机会价值
反复尝试有需求但缺能力主线用户
内疚/失败感在意这一盆生命主线用户
选择易养植物先满足绿色和省心判断是满足，还是妥协
直接换新消费桌面装饰商业侧线
无所谓/放弃低生命附着暂不服务
研究原因走向专业或 DIY非主线

植间先找失败后仍想再养、且已为桌面或植物付费的人。

MVP User

优先寻找人群：五个行为条件的交集

不是“喜欢植物”的人，而是已经尝试过、失败过、仍愿意再试，并且有付费行为的人。

固定桌面在办公桌或学习桌长期停留
失败经历曾经养死或养坏过桌面植物
仍想再养想养一盆会开花的真植物

低精力约束不想增加高频养护负担

Definition

一盆在桌面成长的真实植物系统

活着

补光、底灌、基质、传感降低死亡率。

变好

新叶、花苞、开花形成可见成长。

省心

自动策略和低频维护降低精力消耗。

看见

照片时间线和轻量记录保留变化。

System

系统方案围绕低精力保活设计

保活补光灯环 + 潮汐底灌 + 基质包 + 传感 + 水箱
成长植物策略 + 照片时间线 + 成长记录
省心自动策略 + 低频维护 + 必要提醒
陪伴安静存在 + 克制灯光 + 轻量日记
Validation Layers

四层次分析

RAT

验证目标用户、痛点和付费意愿。

访谈、落地页、价格假门、意向金。

POC

验证补光、潮汐底灌、基质和传感可行性。

先分模块，不追求外观完整。

MVP 样机

少数植物 + 保活系统 + 最小记录。

不做社区、屏幕、泛植物识别。

EVT 前置

漏水、噪音、清洁、续航、传感稳定性。

不急于开模量产。

Decision Gates

五步待验证假设

目标用户找到 5-8 名必须项全中的高匹配用户。
保活弱光桌面对照实验中方案组明显更好。
低精力用户接受维护频率、操作步骤和提醒方式。
成长用户主动关注新叶、花苞、开花时间线。
付费已有桌面消费习惯且有桌面种植潜在需求，有询价行为。
Business Hypothesis

先验证硬件购买，耗材和礼品是后续放大

第一闭环用户是否愿为低精力桌面开花真植物系统付费。
硬件先核算 BOM、装配、物流、售后与渠道成本。
耗材植物包、基质包、缓释肥等复购只在硬件成立后验证。
礼品送礼者与使用者双边需求单独验证。

如果硬件购买不成立，耗材和服务都没有意义。
