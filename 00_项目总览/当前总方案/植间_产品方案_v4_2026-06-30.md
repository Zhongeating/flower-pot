## 零、执行摘要

植间 PlantSpace 是一款面向固定办公桌、学习桌和长期项目桌面的低精力养护型真实植物产品，目标是在桌面上为植物创造一个可长期生长的空间。当前阶段不验证“智能花盆有没有市场”，而验证一个更具体的问题：**已经为桌面和植物付过钱、仍想养一盆会开花真植物的人，是否愿意为桌面弱光保活、可见成长和低精力养护付费。**

本报告将第一阶段目标用户收束为：

> **在固定办公/学习桌长期停留，曾经养死过桌面植物，仍想养一盆会开花或明显成长的真植物，已为桌面绿植或桌搭付过钱，并且不想增加高频养护负担的人。**

这个人群不是泛泛的年轻白领，也不是所有植物爱好者。他们有五个关键行为信号：长期停留在固定桌面，曾经尝试把植物放到桌面，植物死亡或状态变差后仍想再养，已经为桌面环境、绿植或桌搭投入过真实成本，并且不希望植物变成新的高频待办。

植间当前要解决的核心痛点是：用户想把一盆会开花、会变化的真植物长期放在自己最常待的办公/学习桌上，但桌面弱光、养护判断困难、维护动作打断和精力消耗让植物容易死亡或被迫离开桌面。普通绿植太妥协，假花和摆件没有生命变化，现有智能花盆又不够可靠。

因此，植间的产品主张不是“更聪明的花盆”，而是：**一盆能够低精力长期留在桌面、真实生长并被看见的开花植物系统。** 本阶段验证的不是某个单点功能，而是“真实植物 + 可见成长 + 低精力养护”能否在固定桌面场景中同时成立。

当前阶段的关键假设有四个：

| 假设 | 要回答的问题 | 当前状态 |
|------|--------------|----------|
| 目标用户成立 | 是否存在一批行为高度相似、已经付过钱、仍想养开花真植物的人 | 待验证 |
| 保活成立 | 弱光桌面下，补光 + 底灌 + 基质能否显著提高植物存活和状态 | 待实验 |
| 成长价值成立 | 用户是否在意新叶、花苞、开花等可见变化 | 待访谈和样稿测试 |
| 付费成立 | 用户是否愿意为低精力保活和可见成长支付覆盖成本的价格 | 待假门测试 |

在这些假设通过前，植间不扩大植物品类，不做屏幕表情和语音对话，不做复杂社区，不把 AI 日记作为默认功能。产品先证明一件事：**目标用户是否真的需要并愿意购买一盆能低精力留在桌面开花的真植物。**

## 一、目标用户：先验证一个足够尖的人群

### 1.1 MVP 目标用户

植间第一阶段服务的人群定义为：

> **在固定办公/学习桌长期停留，曾经养死过桌面植物，仍想养一盆会开花或明显成长的真植物，已为桌面绿植或桌搭付过钱，并且不想增加高频养护负担的人。**

这句话包含六个筛选条件：

| 条件 | 意义 | 为什么重要 |
|------|------|------------|
| 固定办公/学习桌长期停留 | 用户有稳定桌面场景 | 产品不是家居大盆栽，而是长期陪伴用户的桌面存在 |
| 曾经养死过桌面植物 | 用户遭遇过真实失败 | 保活不是空想功能，而是来自过去损失 |
| 仍想再养 | 植物死亡没有让用户退出 | 说明用户不是一次性装饰消费，而有复购或重试动力 |
| 想养会开花或明显成长的真植物 | 用户期待可见成长和生命变化 | 区分于普通耐阴绿植、假花和摆件 |
| 已为桌面绿植/桌搭付过钱 | 用户有真实付费行为 | 降低“只是觉得有趣”的误判 |
| 不想增加高频养护负担 | 用户有低精力使用约束 | 决定产品不能变成新的待办事项 |

这个定义的价值在于，它不是人口统计标签，而是行为标签。它能指导后续访谈、样机试用、内容测试和价格测试。

### 1.2 第一阶段不服务谁

| 暂不作为 MVP 用户 | 原因 | 对产品的影响 |
|------------------|------|--------------|
| 只想买便宜绿植的人 | 植物死了可以低价替换，不需要高成本系统 | 不以低价花盆竞争 |
| 只需要桌面好看的人 | 假花、香薰、摆件、灯具已经部分满足 | 不把颜值摆件作为主线 |
| 专业植物玩家 | 更愿意 DIY、调参数、换土换肥 | 不优先做深度控制台 |
| 想养任意植物的人 | 早期植物策略无法无限泛化 | MVP 限制植物范围 |
| 只买礼物的人 | 送礼者和使用者分离 | 礼品作为商业侧线单独验证 |
| 只对电子宠物感兴趣的人 | 注意力可能在屏幕和互动，不在真实植物 | 不做屏幕表情作为核心 |

### 1.3 目标用户找到的判定标准

不是能描述一个用户画像，就算找到目标用户。植间需要通过访谈和筛选题验证，是否能稳定找到一批相似的人。

| 判断项 | 达标标准 |
|--------|----------|
| 筛选稳定性 | 连续访谈/招募中，至少找到 5 个以上“必须项全中”的人 |
| 行为一致性 | 他们都曾为桌面绿植、桌搭或养护工具付过钱 |
| 失败经历一致性 | 他们都经历过桌面植物死亡、状态变差、被迫搬离桌面或长期不开花 |
| 重试意愿一致性 | 植物失败后仍想再养，而不是彻底放弃 |
| 需求语言一致性 | 他们能独立说出相似表达：想放桌上、想开花/变化、不想太费精力、不想再养死 |
| 付费信号 | 愿意参与价格测试、留资、预约试用或支付意向金 |

如果只能找到零散兴趣，而找不到一批行为相似的人，植间应回到用户定义重新收束。

## 二、市场入口：真实植物消费 × 桌面体验消费

植间不直接等同于花卉市场，也不直接等同于桌搭市场。它切入的是二者的行为交集：用户既愿意购买真实植物，也愿意为长期停留的桌面环境付费。

### 2.1 真实植物消费足够广泛

据中国花卉协会官网公开的《2026 全国花卉零售业情况分析报告》相关信息，2025 年中国花卉零售市场总规模达到 2490 亿元，同比增长约 13.18%；其中电商渠道 1374 亿元，占比 55.2%，实体花店 1116 亿元，占比 44.8%；2026 年市场规模预计增至 2790 亿元。

这些数据说明，植物消费不是少数兴趣圈层行为。线上购买、即时零售、直播电商、小型盆栽和花卉礼品已经形成成熟交易通路。

但花卉零售市场的大规模并不意味着植间可以直接切入。当前大众花卉消费的主流供给仍偏原始：用户主要购买植物本体、普通花盆、营养土、肥料和低价养护工具，产品形态以低客单价、低系统化程度为主。对植间来说，这既是机会，也是风险：机会在于长期养护系统供给不足，风险在于用户可能已经被低价植物和低价花盆锚定。

因此，这个市场数据不能直接证明智能桌面植物系统成立。它只能说明一件事：**真实植物消费足够广泛，用户并不陌生，但高价值、系统化、可信的长期养护供给仍不充分。**

### 2.2 桌面体验消费提供另一个入口

“桌搭”不是一个统一统计口径的标准行业，因此不能简单写成“2025 年桌搭市场规模 XX 亿”。更稳妥的处理方式，是使用邻近品类和用户行为作为辅助信号：桌上摆饰品、书房家具、显示器灯、显示器支架、键鼠外设、桌面收纳、香薰、摆件、桌面灯具等，都说明用户正在为长期停留的桌面环境投入功能、审美和情绪价值预算。

这些邻近品类对植间的意义不是提供精确 TAM，而是说明：**办公桌和学习桌已经从工具台面变成用户愿意经营的个人空间。**

### 2.3 植间的真实机会在交集

| 市场信号 | 能说明什么 | 不能说明什么 |
|----------|------------|--------------|
| 花卉零售市场 2490 亿元 | 用户买植物、买花卉、买盆栽的行为广泛存在 | 不能证明用户愿为智能花盆付费 |
| 桌面体验消费增长 | 用户愿意为桌面空间的功能、审美和舒适感花钱 | 不能证明用户需要真实植物 |
| 智能花盆差评与评论 | 用户对陪伴、可爱、桌面植物有兴趣，也对不可靠体验敏感 | 不能证明植间方案已成立 |
| 目标用户访谈与假门测试 | 能直接验证“桌面开花真植物 + 低精力保活”的需求 | 当前尚未完成 |

因此，植间不以“智能花盆市场规模”作为立项依据，而以更可验证的交集行为作为起点：**真实植物消费 × 桌面体验消费 × 养护失败 × 低精力需求。**

## 三、场景入口：桌面正在寻找放松、变化和陪伴

办公桌、书桌、自习桌是用户长期停留的位置。这个位置已经有多种替代物：真绿植、假花、摆件、屏保绿植、电子桌宠、鱼缸、微景观、香薰、毛绒物、桌面灯具。它们共同回应一个浅层需求：用户不希望桌面只是工具堆叠，也希望它有一点放松、生命感、变化或陪伴。

真植物相比替代物有三个优势：

| 优势 | 用户感受 |
|------|----------|
| 真实生命 | 它不是模拟内容，而是真的活着 |
| 可见变化 | 新叶、花苞、开花、长高能带来时间感和成就感 |
| 照料结果 | 用户会感觉“这是我养出来的” |

但会开花、会明显变化的真植物在桌面场景里更难成立。许多办公桌和学习桌远离窗户，光照不足；用户不能频繁搬动植物，也不想每天判断浇水、湿度、光照、肥料；维护动作会打断工作和学习，一旦照料变成负担，植物就会从“桌面陪伴”变成“待办事项”。

这就形成了植间要切入的矛盾：

> 用户想要一盆真实、会开花、会变化的植物长期留在自己最常待的桌面上，但桌面环境和低精力使用约束让这件事难以自然成立。

### 3.1 现有桌面替代物只解决了一部分需求

桌面上并不缺“好看”或“可爱”的东西，缺的是能长期低负担存在、持续生长并被用户看见变化的真实生命对象。现有替代方案分别解决了一部分需求，但很少同时覆盖真实、成长和低精力。

| 替代物 | 已解决 | 没解决 |
|--------|--------|--------|
| 普通绿植 | 真实、便宜、容易买 | 弱光下成长有限，变化不明显 |
| 假花/摆件 | 好看、稳定、低维护 | 没有真实生命和成长变化 |
| 电子桌宠/屏保 | 反馈强、可爱、有陪伴感 | 交互智能有限，且不是真实生命 |
| 鱼缸/微景观 | 生命感强、观赏性强 | 维护复杂、风险高、桌面适配难 |
| 情绪智能花盆 | 可爱、互动、传播强 | 保活、传感、续航不足，成长有限 |
| 室内种植机 | 保活强、种植稳定 | 更像专业设备，不够个人桌面化 |

### 3.2 机会在三个价值同时成立

植间第一阶段验证的不是某个单点功能，而是三个价值能否在固定桌面场景中同时成立，并形成付费意愿。

| 价值 | 用户为什么在意 | 对植间的要求 |
|------|----------------|--------------|
| 真实 | 它是真的活着，不是模拟内容 | 产品不能把注意力从植物转移到屏幕 |
| 成长 | 新叶、花苞、开花带来时间感和成就感 | 植物策略和补光必须支持可见变化 |
| 低精力 | 不增加新的高频待办 | 自动养护、低频维护、必要提醒 |

## 四、目标痛点：桌面条件、养护判断和精力消耗共同阻碍体验

植间的目标痛点不是“用户不会养花”这么宽泛，也不是“用户需要情绪陪伴”这么抽象。更准确的痛点是：

> **用户想把一盆会开花、会变化的真植物长期放在自己最常待的办公/学习桌上，但桌面弱光、养护判断困难、维护动作打断和精力消耗让植物容易死亡或被迫离开桌面；普通绿植太妥协，假花和摆件没有生命变化，现有智能花盆又不够可靠。**

简化说，用户不是不想养花，而是想要桌面开花植物，却被弱光、判断困难、卫生风险和精力消耗阻挡。

这个痛点包含四个阻碍。

### 4.1 桌面环境阻碍

| 阻碍 | 具体表现 | 后果 |
|------|----------|------|
| 弱光 | 工位/书桌远离窗户，日照不足 | 开花植物长势弱、不开花、徒长或死亡 |
| 空间有限 | 桌面要放电脑、书、本子、外设 | 植物必须体积克制、维护动作不能占空间 |
| 卫生要求 | 土表潮湿、虫害、水渍会影响桌面 | 用户不愿让植物破坏工作/学习环境 |
| 维护距离 | 放窗边更适合植物，但离开用户常驻位置 | 植物活着却不再参与桌面生活 |

### 4.2 养护判断阻碍

用户不是完全不愿意照料植物，而是不知道什么时候该做什么。浇多会烂根，浇少会干死；光照不足时，用户不知道是植物品种问题、位置问题还是养护问题。信息多但执行难，教程并不能稳定解决问题。

### 4.3 维护打断阻碍

桌面植物如果需要频繁搬动、浇水、擦水、查教程、换位置、观察状态，就会打断用户的工作和学习节奏。对目标用户来说，植物的价值是“低负担地在身边变化”，不是增加一个高频待办。

### 4.4 精力消耗阻碍

这是低注意力交互和陪伴设计的基础。用户希望植物有存在感，但不希望它要求持续关注、频繁操作或制造心理负担。低精力消耗不是“不参与”，而是：

| 用户希望 | 产品含义 |
|----------|----------|
| 不用每天判断浇水 | 系统自动完成基础供水，用户只处理低频维护 |
| 不用理解复杂参数 | 不把 PPFD、EC、湿度曲线直接抛给普通用户 |
| 不被频繁提醒打扰 | 状态反馈克制，只在必要时提示 |
| 不因为植物状态焦虑 | 系统先保证植物稳定，再提供轻量记录 |
| 偶尔看见它在变好 | 用新叶、花苞、开花和照片时间线形成正反馈 |

因此，植间后续的“低注意力陪伴”不应从屏幕表情或语音互动开始，而应建立在低精力养护之上：植物稳定活着、慢慢变好、用户偶尔看见变化，而不是被设备不断召唤。

## 五、需求定义：低精力的桌面开花真植物保活系统

植间当前验证的具体需求可以表述为：

> **我需要一盆能长期留在办公/学习桌上、不用我天天判断养护、还能开花或明显成长的真植物。**

这句话比“我需要智能花盆”更接近用户需求，因为它包含用户想要的结果、使用场景和约束。

如果转成用户语言，这个需求也可以表述为：**我想省心地在桌上养一盆会变化、最好会开花的真植物。**

| 需求要素 | 对应含义 | 产品回应 |
|----------|----------|----------|
| 长期留在办公/学习桌上 | 场景固定，不能依赖窗台和阳台 | 桌面化尺寸、后背水箱、开放结构 |
| 不用天天判断养护 | 低精力、低认知负担 | 自动补光、低水位潮汐底灌、基质包、基础传感 |
| 会开花或明显成长 | 不只活着，还要产生变化 | 植物策略、补光参数、成长记录 |
| 真植物 | 价值来自真实生命，而非屏幕模拟 | 不以电子宠物为核心 |
| 低打扰陪伴 | 有存在感但不抢注意力 | 低注意力灯光、照片时间线、必要提示 |

### 5.1 需求成立的判定标准

| 判断项 | 不成立的表现 | 成立的表现 |
|--------|--------------|------------|
| 用户能复述 | “挺有意思”“看起来好玩” | “我就是想放桌上，但之前养死了，还想养会开花的” |
| 有现有支出 | 只是点赞、收藏、口头喜欢 | 买过绿植、花盆、补光灯、营养液、桌搭用品 |
| 替代方案不满意 | “绿萝就够了”“死了再买” | “绿萝活是活，但没变化；开花的又养不住” |
| 低精力约束明确 | 愿意每天照料、研究参数 | 希望系统自动处理基础养护，只做低频维护 |
| 付费意愿真实 | 只接受很低价格或免费试用 | 愿意留资、预约试用、接受价格测试或交意向金 |

## 六、竞品与替代品：情绪互动能吸引用户，但不是长期吸引力

现有方案分别解决了真实、稳定、记录、装饰或情绪的一部分，但很少同时覆盖真实开花植物、桌面保活、可见成长和低精力陪伴。

| 类型 | 代表 | 已解决的需求 | 留下的缺口 |
|------|------|--------------|------------|
| 普通绿植 | 绿萝、白掌、多肉、豆瓣绿 | 真实、便宜、容易购买 | 弱光下成长有限，开花难，死亡风险高，缺少状态反馈 |
| 开花盆栽 | 非洲堇、栀子、长寿花、小型绣球等 | 真实、有花、有变化 | 桌面弱光和养护判断门槛更高 |
| 假花/摆件 | 仿真花、桌面装饰、香薰摆件 | 稳定、好看、低维护 | 无生命变化，长期新鲜感弱 |
| 桌搭工具 | 灯具、支架、收纳、键鼠、摆件 | 改善桌面审美和效率 | 不提供真实生命和成长结果 |
| 情绪智能花盆 | Ivy/Ivy2 等 | 可爱、互动、送礼传播 | 保活、传感器、续航、系统稳定性存在风险 |
| 成熟室内种植设备 | Click & Grow、AeroGarden | 保活强、种植稳定 | 偏厨房/家庭种植设备，桌面审美和个人陪伴属性不足 |

Ivy/Ivy2 是最接近的情绪智能花盆样本。基于此前整理的 103 条用户评论，差评集中在水位或湿度传感器失灵、换植物后检测失效、系统卡死或配网失败、续航短、光照不足、封闭结构导致植物死亡等问题。其中约 11.7% 的评论主动质疑“伪智能”“花盆加 AI”“骗人”，说明低质体验正在损害品类信任。

这些反馈给出两个判断：

1. 用户并不排斥桌面植物的情绪价值，“陪伴”“可爱”“桌面搭子”“电子宠物”等表达说明这个方向有吸引力。
2. 情绪价值必须建立在真实植物长期存活和设备可靠性之上。植物死掉、传感器失灵、设备频繁出问题时，用户会把产品从“陪伴”归类为“智商税”。

因此，植间必须先交付可信的低精力保活和可见成长，再谈陪伴表达。

情绪互动可以带来第一眼吸引，但长期吸引力仍依赖植物真实状态、成长变化和设备可靠性。植间需要避免把产品推向电子玩具心智，而应让真实植物成为长期内容源。

---

## 七、产品定义：一盆会照顾自己的桌面开花植物

植间的产品定义是：

> **一盆能在室内桌面稳定生长、低精力维护、产生可见花叶变化，并被轻量记录的真实植物系统。**

植间交付的是一盆真实植物，以及围绕这盆植物建立的低维护养护系统和成长记录体验。植物是长期内容源，花盆是基础设施，算法和 AI 是后台能力。用户不需要理解光照参数、灌溉策略、基质配比和传感器读数，只需要获得四个结果：植物活着，植物在变好，用户觉得省心，变化被看见。

| 定义层 | 说明 | 对用户的直接含义 |
|--------|------|------------------|
| 活着 | 补光、底灌、基质和传感降低失败率 | 不再每天担心该不该浇水、要不要搬去晒太阳 |
| 变好 | 选择适合桌面的植物策略，让新叶、花苞、开花成为可见结果 | 用户能看到“它真的在变好” |
| 省心 | 基础养护自动化，维护动作低频且简单 | 植物不变成新的待办负担 |
| 看见 | 照片时间线、人工标记、轻量日记样稿 | 变化能被保存、回看和分享 |

低注意力陪伴是上述结果成立后的体验表达：植物安静存在，必要时提醒，成长变化被轻量记录，而不是用屏幕表情或高频语音不断召唤用户。

### 7.1 产品能力对应用户任务

| 用户任务 | 产品能力 | 说明 |
|----------|----------|------|
| 不再养死 | 顶部补光灯环、低水位潮汐底灌、混配基质包、多高度电容检测、后背水箱 | 解决弱光、忘浇水、误浇水和土壤闷湿问题 |
| 不太费心 | 自动策略、必要提醒、低频维护、基础状态呈现 | 降低用户判断成本和维护打断 |
| 看到变化 | 植物品类策略、照片时间线、人工记录新叶/花苞/开花 | 让成长成为持续价值 |
| 安静陪伴 | 低注意力灯光、无屏幕形态、对照日记样稿 | 降低打扰，保留桌面存在感 |

### 7.2 功能克制原则

植间当前不把屏幕、表情和语音对话作为核心。原因不是这些功能一定没有价值，而是它们容易把注意力从植物转移到设备本身，也会增加成本、功耗、可靠性和用户期待管理难度。

低注意力交互的起点应当是：

- 植物状态稳定时，不打扰用户。
- 需要用户处理时，提示清楚且低频。
- 成长变化通过照片和记录呈现，而不是即时打断。
- 灯光优先服务植物补光，其次才承载情绪反馈。

## 八、MVP 范围：四层次验证，而不是一次做完整样机

硬件产品不应一开始就把完整样机当作 MVP。植间应把验证拆成 RAT、POC、MVP 样机和 EVT 前置四个层次，让每一层只回答对应阶段最危险的问题。

### 8.1 四层次分析

| 阶段 | 目的 | 做什么 | 不做什么 |
|------|------|--------|----------|
| RAT 最危险假设测试 | 验证目标用户、痛点和付费意愿 | 用户访谈、筛选问卷、落地页、价格假门、照片时间线/日记样稿 | 不做完整硬件，不承诺量产形态 |
| POC 模块验证 | 验证技术可行性 | 补光、潮汐底灌、基质、传感、水箱分模块实验 | 不追求外观完整，不做 App 完整体验 |
| MVP 样机 | 验证真实桌面闭环 | 少数植物 + 保活系统 + 最小状态查看 + 成长记录 | 不做社区、排行榜、屏幕表情、泛植物识别 |
| EVT 前置 | 验证工程指标 | 漏水、噪音、清洁、续航、传感稳定性、异常恢复 | 不急于开模量产 |

### 8.2 MVP 必须验证的闭环

| 闭环 | 验证问题 | 最小实现 |
|------|----------|----------|
| 目标用户闭环 | 是否能稳定找到这类用户 | 筛选表 + 10-15 人访谈 |
| 保活闭环 | 弱光桌面下能否活得更好 | 补光 + 底灌 + 基质 + 对照实验 |
| 低精力闭环 | 用户是否觉得养护负担明显降低 | 低频维护任务 + 必要状态提示 |
| 成长闭环 | 用户是否在意开花、新叶、时间线 | 照片样稿 + 实验记录 + 概念测试 |
| 付费闭环 | 用户是否愿意为覆盖成本的体验付费 | 价格假门、意向金、试用报名 |

### 8.3 MVP 暂不包含

| 暂不做 | 原因 |
|--------|------|
| 屏幕表情和语音对话 | 容易把产品推向电子玩具心智，增加成本和注意力占用 |
| 复杂社区、排行榜、金币体系 | 与低精力、低注意力桌面陪伴方向不一致 |
| 大规模植物识别和推荐 | 当前 MVP 先服务少数植物策略，避免泛化过早 |
| App 订阅 | 用户价值和付费意愿尚未验证 |
| 多设备管理 | 当前 MVP 先验证单盆闭环 |
| 全自动 AI 日记 | 先用样稿验证价值，再决定是否自动化 |
| 任意植物兼容 | 早期植物策略必须可控，否则保活实验失真 |

## 九、证据体系与风险

当前证据足以支持继续探索，但不足以支持完整样机和商业化投入。v4 的证据体系需要明确区分事实、强假设、弱假设和待验证项。

### 9.1 证据分级

| 证据等级 | 当前主张 | 支撑材料 | 判断 |
|----------|----------|----------|------|
| 已验证事实 | 情绪智能花盆存在系统稳定性和保活风险 | Ivy/Ivy2 103 条评论，集中出现传感器失灵、续航差、光照不足、系统卡死等反馈 | 可作为竞品缺口依据 |
| 已验证事实 | 桌面弱光会限制植物长期生长 | 竞品评论与用户访谈均出现办公室光线差、工位养不活等问题 | 可作为保活底座依据 |
| 已验证事实 | 低质产品会触发“伪智能”“智商税”质疑 | Ivy/Ivy2 评论中约 11.7% 主动质疑品类真实性 | 可作为克制智能表达依据 |
| 强假设 | 存在愿为“桌面开花真植物 + 低精力保活”付费的人群 | 画像、评论、桌搭行为有支持，但未完成定向筛选 | 需通过目标用户访谈验证 |
| 强假设 | 保活系统能显著改善桌面植物体验 | 养护方案有工程决策链，但缺真实桌面长期实验 | 需通过种植对照实验验证 |
| 强假设 | 成长变化比单纯存活更能形成持续价值 | 用户语言和种植经验有支持，但样本不足 | 需通过时间线样稿和访谈验证 |
| 强假设 | 低精力养护是目标用户接受产品的关键约束 | 用户希望陪伴但不希望负担，符合桌面场景逻辑 | 需通过概念测试和试用观察验证 |
| 弱假设 | 高压人群能自然理解植物成长与自身成长的关联 | 当前仍是意义迁移假设 | 只作为后续内容表达测试 |
| 待验证 | 用户愿为覆盖硬件、装配、售后与渠道成本的价格付费 | 当前无完整 BOM 和真实支付数据 | 需先做成本核算，再做假门测试 |

### 9.2 核心风险

| 风险 | 严重性 | 当前处理 |
|------|:--:|----------|
| 找不到足够尖的目标用户 | 致命 | 先做筛选访谈，不急于样机开发 |
| 植物养不活或不开花 | 致命 | 先做 3 个月弱光桌面对照实验 |
| 用户只需要好看和省事 | 高 | 访谈中区分装饰替换型用户与生命附着型用户 |
| 低精力需求与真实植物养护冲突 | 高 | 明确自动化边界和低频维护任务 |
| 成长变化价值不足 | 高 | 展示新叶、花苞、开花时间线，观察用户反应 |
| 价格接受度不足 | 高 | 完成 BOM 和价格假门测试后再决定样机投入 |
| 品类信任不足 | 高 | 避免过度智能承诺，优先展示植物真实状态和保活能力 |
| 耗材履约复杂 | 中 | 当前 MVP 限制植物和基质范围 |

## 十、验证计划与决策门槛

当前验证应优先杀掉最危险假设，而不是证明所有功能都值得做。

### 10.1 五步待验证假设

| 假设 | 验证方式 | 继续标准 |
|------|----------|----------|
| 目标用户 | 筛选问卷 + 访谈 | 找到 5-8 名必须项全中的高匹配用户 |
| 保活 | 弱光桌面对照实验 | 方案组明显优于普通盆栽 |
| 低精力 | 维护任务样稿 + 概念测试 | 用户接受维护频率、操作步骤和提醒方式 |
| 成长 | 新叶、花苞、开花时间线访谈 | 用户主动关注并愿意记录 |
| 付费 | 市场验证、询价或假门测试 | 已有桌面消费习惯且有桌面种植潜在需求，有询价行为 |

### 10.2 第一阶段验证计划

| 风险 | 实验 | 样本/周期 | 继续标准 |
|------|------|-----------|----------|
| 目标用户是否存在 | 筛选问卷 + 深访 | 招募 20-30 人，筛出 5-8 名高匹配用户 | 高匹配用户能稳定出现，且语言和行为高度相似 |
| 痛点是否成立 | 问题访谈，不先讲方案 | 10-15 名桌面绿植/桌搭付费用户 | 用户主动说出弱光、养死、想开花、低精力等关键问题 |
| 保活是否成立 | 弱光桌面对照种植 | 3-5 种植物 × 每种 3 盆，周期 3 个月 | 植间方案组存活率、长势或开花表现显著优于对照组 |
| 低精力是否成立 | 维护任务概念测试 | 与深访合并 | 用户认为维护频率、操作步骤和提醒方式可接受 |
| 成长是否有价值 | 新叶/开花时间线访谈 | 8-10 名高匹配用户 | 用户主动表达想记录、想拥有或愿为成长变化付费 |
| 价格是否成立 | 成本核算后的价格假门测试 | 小红书/落地页/意向金 | 至少一个可覆盖成本的价格带出现有效点击、咨询或意向金行为 |

### 10.3 目标用户筛选表

| 筛选条件 | 必须/加分 | 判定方式 |
|----------|-----------|----------|
| 固定办公/学习桌每天停留 4 小时以上 | 必须 | 访谈/问卷 |
| 过去 12 个月买过桌面绿植或桌搭用品 | 必须 | 订单、照片、用户描述 |
| 曾经养死过或养坏过桌面植物 | 必须 | 具体经历复述 |
| 死后仍想再养 | 必须 | 复购/收藏/搜索/表达意愿 |
| 想养会开花或明显成长的真植物 | 必须 | 植物偏好提问 |
| 现有替代方案不满意 | 必须 | 对绿萝、假花、普通花盆、智能花盆的评价 |
| 愿意参与价格测试或试用 | 必须 | 留资、试用报名、意向金 |
| 有桌面审美/记录/分享习惯 | 加分 | 桌面照片、小红书收藏、桌搭消费 |
| 有低精力养护诉求 | 加分 | 明确说“不想天天管”“不想增加待办” |

### 10.4 决策门槛

| 结果 | 决策 |
|------|------|
| 找不到高匹配目标用户 | 暂停样机开发，重做用户定义 |
| 目标用户存在，但只想低价装饰 | 收缩为桌面绿植/耗材方向，暂缓智能硬件 |
| 保活实验失败 | 暂停情绪功能扩展，回到养护系统重构 |
| 保活成立但成长价值弱 | 产品收缩为低维护桌面植物，削弱开花和记录叙事 |
| 保活和成长成立，但低精力体验不成立 | 简化维护流程，重做交互和水箱/清洁设计 |
| 用户只接受无法覆盖成本的低价 | 重新评估 BOM、产品定位和目标用户 |
| 目标用户、保活、成长、付费均成立 | 进入样机迭代和小规模用户试用 |

## 十一、商业假设与后续路径

植间短期不追求大规模商业化，先验证用户价值和第一商业闭环。第一商业闭环不是耗材，也不是 AI 服务，而是用户是否愿意为一套低精力桌面开花真植物系统支付硬件价格。硬件购买不成立时，耗材、礼品和增值服务都只是推演。

### 11.1 收入假设

| 收入来源 | 内容 | 当前状态 |
|----------|------|----------|
| 硬件购买 | 花盆本体、灯环、水箱、泵、传感器、基础结构 | 第一商业闭环，必须优先验证 |
| 植物与耗材包 | 植物、基质包、缓释肥、苔藓铺面、植物补充包 | 需验证复购频率和履约成本 |
| 礼品套装 | 花盆 + 植物 + 基质 + 卡片/成长寓意包装 | 送礼者与使用者分离，单独验证 |
| 渠道测试 | 桌搭、绿植、备考、礼品场景小规模测试 | 用于验证获客效率和场景匹配度 |
| 增值服务 | AI 日记、植物策略包、扩展植物包 | 暂不进入 MVP 商业模型 |

### 11.2 成本与价格验证

价格测试必须建立在完整成本核算之后，至少包括：

| 成本项 | 需要核算的内容 |
|--------|----------------|
| 硬件 BOM | 补光灯环、水箱、泵、传感器、结构件、控制板、电源等 |
| 结构与制造 | 打样、开模、装配、密封、防水、水路可靠性测试 |
| 植物与耗材 | 植物、基质包、缓释肥、包装损耗 |
| 履约 | 物流、植物运输损耗、售后换货、客服支持 |
| 渠道 | 平台佣金、投流、内容制作、礼品包装溢价 |

价格测试不能只问“你觉得多少钱合适”。更有效的方式是给出具体产品承诺和价格带，观察用户是否愿意点击、咨询、留资、预约试用或支付意向金。
