# 植间智能花盆浸盆式潮汐灌溉技术决策报告

> 日期：2026-06-20  
> 对象：20 cm 直径桌面智能花盆，底部水泵注水/抽水，实现浸盆式潮汐灌溉  
> 结论等级：可以作为 MVP 主方案，但必须以“标准化基质 + 短时底部供水 + 完全抽回游离水 + 周期冲盐 + 通风干湿循环”为边界，不能做成长期坐水、自吸盆或简化版水培。

---

## 1. 结论先行

对于“20 cm 直径、桌面、封闭底部水路、用水泵从底部注水和抽水”的智能花盆，采用浸盆式潮汐灌溉是合理的，而且比单纯顶部浇水、自吸盆、连续水培、滴灌、喷雾更符合你的产品目标。

但它成立的逻辑不是“底部潮汐一定比顶部浇水更不湿”。在同一种基质、同一盆型、同样充分湿润并排掉游离水后，盆土最终接近的含水状态主要由基质孔隙、盆高和排水边界决定，而不是由水从上方还是下方进入决定。潮汐法的产品价值在于：它能把浇水变成可测量、可抽回、可记录、可和通风/补光联动的水分事件。

但这个结论成立有前提：

1. 它必须是“潮汐式”，不是“坐水式”。水只在浇水事件中进入盆底，浸泡后必须抽回。
2. 它必须允许根区重新获得空气。每次灌溉后，基质要进入可测量的干回落过程。
3. 它必须有冲盐策略。长期只从底部供水，会把肥盐/矿物盐留在介质中，不能完全替代顶部浇透。
4. 它必须做水箱卫生管理。封闭水箱不等于干净水箱，营养水、根系残渣、光照和温度都会带来藻类/细菌/病原风险。
5. 它必须绑定标准化基质。泵能抽回的是水腔里的游离水，不是基质孔隙中的毛细水；基质才决定真正的持水量、通气性和干回落速度。
6. 它不能宣称适配所有植物。多肉、球根、食虫植物、兰科附生、水草/水陆缸植物要单独规则；其中部分喜干植物可进入“标准颗粒基质 + 低水位短潮汐”的验证分支，球根休眠期必须进入停水规则。

一句话判断：

> 你的方案不是“万能浸盆法”，而是一个小型化、单盆隔离的 ebb-and-flow subirrigation 系统。这个方向专业上站得住，但成败取决于基质标准化、抽回游离水、通风干回落、冲盐、卫生和植物规则，而不是“泵能不能把水打进去”。

---

## 2. 本次判断的核心问题

产品要解决的不是“如何浇水”这么窄的问题，而是：

```text
目标用户想要桌面植物长期活着
  -> 智能花盆要减少新手误操作
  -> 浇水系统要避免过干、过湿、闷根、传感器误导
  -> 同时要适合桌面环境：干净、少虫、少水渍、可自动化
```

所以技术选择必须同时满足五个约束：

| 约束 | 含义 | 对 20 cm 封闭花盆的影响 |
|---|---|---|
| 植物生理 | 根需要水，也需要氧气 | 不能长期让根处于饱和水环境 |
| 桌面场景 | 不能漏水、脏桌面、滋生虫 | 底部封闭水路有优势，但水箱卫生变重要 |
| 用户体验 | 用户不懂见干见湿，且不想频繁操作 | 自动注排水优于频繁手浇 |
| 硬件实现 | ESP32-S3 + 水泵 + 水箱 + 风扇可控 | 潮汐循环可被程序化 |
| 可验证性 | MVP 必须能做 A/B 实验 | 可测水量、浸泡时间、抽回量、重量变化、存活率 |

## 3. 灌溉方式全景对比

### 3.1 家庭盆栽/容器植物常见方式

| 方式 | 原理 | 优点 | 缺点 | 对本方案适配度 |
|---|---|---|---|---|
| 顶部浇透 | 从基质表面浇水，水从底孔排出 | 能冲盐；符合传统盆栽；简单 | 桌面易漏水；土表湿，可能小黑飞；用户容易半截水/过浇 | 中，适合作为维护/冲盐，不适合作为唯一自动方案 |
| 少量顶部补水 | 用户少量频繁浇 | 操作简单 | 最容易浇不透，根区湿度不均 | 低，不建议作为智能方案 |
| 底部浇水/浸盆 | 水从底部被介质毛细吸上来 | 叶面干燥；小盆吸水均匀；桌面干净 | 不冲盐；浸泡过久会缺氧；需倒掉余水 | 高，但必须短时并抽回 |
| 自吸盆/棉绳供水 | 储水层通过棉绳或介质连续毛细供水 | 便宜、无泵、适合低成本懒人盆 | 长期偏湿；难做干湿循环；盐分和藻类风险 | 中低，不适合“养不死”的高置信 MVP |
| 毛细垫 | 盆底接触湿垫，持续吸水 | 温室/育苗统一供水，省人工 | 垫子常湿，藻类/病害/盐分管理复杂 | 低，桌面单盆没必要 |

### 3.2 温室/农业灌溉方式

| 方式 | 原理 | 优点 | 缺点 | 对本方案适配度 |
|---|---|---|---|---|
| Subirrigation 底灌 | 从容器底部供给水/营养液 | 水肥利用率高；叶面干；均匀 | 循环水管理、盐分、病害风险 | 高，是理论母体 |
| Ebb and flow 潮汐灌溉 | 定时灌入托盘/槽，短时浸泡后排回 | 可自动化；可测水位/时间；兼顾吸水和排水 | 对水箱卫生、排水速度、介质毛细性有要求 | 最高，与你方案最接近 |
| Flood floor 漫灌地面 | 温室地面整体进水排水 | 大规模省人工 | 需要大型平整地面和储液池 | 低，仅作原理参考 |
| 滴灌 | 滴头把水送到每株基部 | 精准、节水、成熟 | 小盆易局部湿润；滴头堵塞；管路复杂 | 中，适合大盆/菜园，不是 20 cm 桌面首选 |
| 顶喷/喷灌 | 喷头从上方喷水 | 覆盖大面积，降温 | 叶面湿，病害/水渍风险；桌面不友好 | 低 |
| 喷雾/雾化 | 细雾提高湿度或给插穗保湿 | 适合扦插、雨林缸、空气湿度 | 不是稳定根区供水；易霉/水渍 | 低，只作辅助湿度 |

### 3.3 水培/半水培系统

| 方式 | 原理 | 优点 | 缺点 | 对本方案适配度 |
|---|---|---|---|---|
| DWC 深水培 | 根部分或大部分悬入营养液 | 生长快，适合叶菜 | 要控 pH/EC/氧气；根腐和藻类风险 | 低，不建议作为普通桌面绿植 MVP |
| Kratky 被动水培 | 靠水面下降形成空气根区 | 无泵、低成本 | 水位和作物周期要求高；更适合短周期叶菜 | 中低，只适合叶菜/猫草等短周期 |
| NFT 营养液膜 | 浅层营养液持续流过根区 | 高效、集中管理 | 依赖泵，断电快干；通道系统复杂 | 低，不适合单盆观叶花盆 |
| 滴灌水培 | 营养液滴到基质/根区，回收或不回收 | 适合果菜/商业生产 | 管路和营养管理复杂 | 中低 |
| Aeroponics 气雾培 | 根悬空，周期喷营养雾 | 氧气足，技术感强 | 高故障风险，喷头堵塞，成本高 | 低，不适合 MVP |
| 半水培/LECA | 惰性颗粒 + 储水层 | 干净、少土虫、观感好 | 换根期风险；EC/pH/盐分管理；不适合所有植物 | 中，可作为后续植物/介质分支 |

## 5. 20 cm 封闭花盆的物理约束

假设花盆直径 20 cm，则底面积近似：

```text
半径 = 10 cm
底面积 = π × 10² ≈ 314 cm²
```

如果整个底部平面进水：

| 水深 | 约等于水量 |
|---:|---:|
| 0.5 cm | 157 mL |
| 1.0 cm | 314 mL |
| 1.5 cm | 471 mL |
| 2.0 cm | 628 mL |
| 2.5 cm | 785 mL |

UMass 温室 ebb-flow 的 3/4-1 inch 水深约等于 1.9-2.54 cm。放到 20 cm 圆形底面，理论水量约 600-800 mL。但桌面花盆不应直接照搬温室参数，因为：

- 20 cm 花盆基质高度、内盆结构、底孔面积不同。
- 赤玉土/颗粒基质的毛细高度和吸水速度不同于泥炭基质。
- 封闭花盆没有自然排水到外部，所有多余水都必须可靠抽回。
- 桌面植物的根区更小，更容易因为残留水形成长期湿脚。

因此，20 分钟浸泡不应作为默认定值。它只能作为第一版实验参数之一。

更合理的 MVP 参数路线：

| 参数 | 建议起点 | 验证方式 |
|---|---|---|
| 注水水深 | 0.8-1.5 cm 起步，而不是直接 2.5 cm | 看基质上层是否均匀变重，不追求土表全湿 |
| 单次水量 | 250-500 mL 起步，按内盆有效底面积修正 | 称重：注水前、浸泡后、抽回后 |
| 浸泡时间 | 5、10、15、20 min 做阶梯实验 | 看吸水量曲线是否趋于平台 |
| 抽回标准 | 抽到水位低于进水口/回水口安全线 | 观察 24h 后盆底是否仍有游离水 |
| 干回落间隔 | 不按固定天数，按盆重/湿度/植物规则触发 | 建立每种植物的 dry-back 曲线 |

## 6. 容器土壤水模型：上浇和潮汐的真实差异

判断浸盆式潮汐灌溉是否合理，不能只看“水从上来还是从下去”，而要看容器基质里的水最终以什么形态存在。

盆栽基质中的水大致分三类：

| 水分形态 | 位置 | 能否被泵抽走 | 对植物的意义 |
|---|---|---|---|
| 游离水/重力水 | 水腔、大孔隙、能自由流动的水 | 能直接或间接排走 | 长时间存在会挤占空气，导致坐水风险 |
| 毛细水 | 被基质孔隙吸住的水 | 不能靠普通水泵抽走 | 主要可用水来源，也可能导致基质长时间偏湿 |
| 吸湿水 | 贴附在颗粒表面的薄水膜 | 不能抽走 | 多数植物难以直接利用 |

这意味着：

```text
泵抽回 = 清空底部水腔/托盘中的游离水
不等于 = 把基质里的毛细水抽干
```

顶部浇透和潮汐底灌如果都做到“充分湿润 + 排掉游离水”，一段时间后都可能接近同一种容器持水状态。盆底都会比上部更湿，这是容器栽培中的悬挂水层现象，不是潮汐法独有的问题。

两者真正不同的是湿润路径和产品可控性：

| 维度 | 顶部浇透 | 浸盆式潮汐灌溉 |
|---|---|---|
| 湿润路径 | 水靠重力从上到下穿过根区 | 水靠毛细作用从下向上进入基质 |
| 根区到水确定性 | 高，粗颗粒土也能被水流穿透 | 取决于基质毛细能力，粗颗粒可能吸不上去 |
| 冲盐能力 | 强，可把盐分向下带出 | 弱，必须周期顶部冲盐或取出内盆冲洗 |
| 土表状态 | 容易湿，虫害和藻类风险上升 | 可保持相对干燥，桌面和防虫体验更好 |
| 自动化可控 | 可做，但要处理喷淋、水渍和漏水 | 水位、时间、抽回量更容易封闭控制 |
| 排水边界 | 依赖底孔和用户倒掉托盘水 | 系统主动抽回水腔游离水 |

所以，不能用“潮汐底灌会让底部湿，而顶部浇水不会”来论证本方案。更准确的逻辑是：

```text
同一种基质充分湿润后，最终持水量主要由基质和盆型决定；
潮汐法的优势不是让基质天然更干，
而是让水分事件可测、可控、可抽回、可记录，并能和补光/导风形成闭环。
```

这也推导出一个产品边界：如果基质不标准化，潮汐法会变得不可预测。泥炭/椰糠吸水快但可能保水太久；粗颗粒多肉土通气好但可能毛细上升不足；半无机中颗粒基质最可能成为 MVP 的主基质窗口。

## 7. 为什么不是其他方案

### 7.1 不选纯顶部浇水作为主方案

顶部浇水最大的优点是湿润根区确定、能冲盐，这是它必须保留为维护动作的原因。但作为自动主浇水方案，它有几个产品问题：

- 桌面场景容易漏水、溅水、弄脏桌面。
- 土表长期湿，容易小黑飞和藻类。
- 顶部喷淋会打湿叶片和花朵，增加水渍/病害/观感问题。
- 用户最常见的问题不是没有水，而是不知道是否浇透、是否过量。

所以顶部浇水适合做“周期冲盐/维护模式”，不适合作为桌面 MVP 的唯一日常自动浇水主方式。这个判断不是因为顶部浇水更容易烂根，而是因为它在桌面清洁、防虫、自动化封闭控制和用户误操作上不如潮汐方案。

### 7.2 不选自吸盆/棉绳供水作为主方案

自吸和毛细绳便宜、安静、无泵，但它的核心问题是连续供水：

- 很难形成明确的干湿循环。
- 不容易根据植物生命周期动态停水。
- 盐分累积更难处理。
- 对“养不死”的差异化不够，容易和普通懒人盆混同。

你的产品要证明“会养自己的植物”，而不是做一个更贵的懒人盆，所以不建议用自吸作为核心方案。

### 7.3 不选纯水培/半水培作为主方案

水培和半水培可以很干净，也适合绿萝、富贵竹、猫草、生菜等一部分植物。但如果作为通用智能花盆主方案，会马上引入：

- pH/EC 监测与营养液管理。
- 根系缺氧与水温问题。
- 藻类、细菌、换水、清洗问题。
- 植物转水培换根失败。
- 用户想种“自己的植物”时适配范围变窄。

所以水培适合做后续模式或特定植物包，不适合作为第一版通用桌面植物方案。

### 7.4 不选滴灌作为主方案

滴灌在农业和温室里非常成熟，但 20 cm 桌面单盆里问题明显：

- 需要滴头/管路，结构复杂且可能堵塞。
- 单点滴灌容易局部湿、局部干。
- 土表仍然湿，不利于你想要的防小黑飞叙事。
- 相比底部注排水，用户可感知差异不强。

滴灌适合多盆、大盆、果菜和阳台系统，不是这个桌面 MVP 的最优主方案。

### 7.5 不选喷雾/雾化作为主方案

喷雾适合提高空气湿度、扦插生根、雨林缸或空气凤梨辅助，但它不是根区稳定供水方案。作为主方案会带来水渍、霉菌、叶面病害和桌面潮湿问题。

---

## 8. 为什么浸盆式潮汐灌溉适合你的方案

### 8.1 它与产品核心差异一致

你的差异化是：

```text
浸盆式潮汐灌溉 + 顶部补光 + 开放结构 + 主动导风
```

这个组合的逻辑是完整的：

```text
底部短时供水
  -> 根区能吸到水
  -> 抽回后不长期坐水
  -> 土表相对干燥
  -> 风扇带走表层湿气
  -> 降低闷湿、小黑飞、霉菌风险
  -> 补光解决工位低光
  -> 植物更可能活得久
```

这里的关键不是“浸盆”两个字，也不是假设它比顶部浇水天然更干，而是“水分事件可控 + 根区干湿循环 + 表层通风”。

### 8.2 它比 Ivy2 式错误浇水更容易做成确定动作

竞品 Ivy2 的问题之一是传感器不准导致用户做错误动作。潮汐灌溉可以把浇水动作标准化：

```text
注入固定水位
等待固定或动态时间
抽回多余水
记录实际吸水量
等待干回落
```

这比“传感器说渴了，用户从上面浇一点”更可控。

### 8.3 它适合 20 cm 单盆封闭水路

单盆封闭水路比多盆共用水槽更容易控制病原传播。只要水箱可拆洗、避光、定期换水，就能比温室多盆循环系统降低交叉感染风险。

同时，20 cm 直径足够容纳：

- 独立内盆/种植篮。
- 底部浅水腔。
- 回水口/抽水口。
- 水位传感或浮子结构。
- 简单导风通道。

## 9. 必须注意的风险与设计要求

### 9.1 风险一：长期坐水

这是第一优先级风险。UMD Extension 明确说 bottom watering 后要倒掉多余水，不能让 houseplants sit in water。

设计要求：

- 抽水动作必须是默认闭环，不是可选动作。
- 系统要确认抽回成功，例如水位低于安全线，或泵运行后无可抽水。
- 断电时不能让内盆泡在满水位里。
- 花盆底部要设计“最高安全水位”和“残留水死角”排查。

### 9.2 风险二：盐分累积

长期底部供水不会像顶部浇透那样把盐分向下冲出。UMD 和 PSU 都强调可溶盐会导致萎蔫、叶尖焦枯、根尖死亡等问题。

设计要求：

- MVP 可以不内置自动顶部冲洗，但必须在产品说明和 App/Agent 规则中提示周期冲盐。
- 如果使用营养液，EC 监测优先级上升。
- 如果不做 EC，MVP 应避免自动投肥，只做清水潮汐灌溉 + 缓释肥/低肥策略。
- 结构上最好允许用户取出内盆，在水槽中顶部浇透冲洗。

### 9.3 风险三：封闭水箱病原与藻类

APS 和 UMass 的资料都指向同一个事实：循环水/储水系统不是天然安全的。

设计要求：

- 水箱避光，减少藻类。
- 水箱可拆洗，而不是永久封死。
- 回流水不要跨盆共享。
- 每次抽回后尽量减少游离水与有机残渣长时间接触。
- 需要换水提醒，例如按天数、灌溉次数或水箱温度估计。

### 9.4 风险四：基质毛细能力不稳定

赤玉土、颗粒土、泥炭、椰糠、珍珠岩、水苔、树皮的毛细吸水完全不同。20 分钟对一种介质合适，对另一种可能过长或无效。

这不是潮汐法的边缘问题，而是主问题。泵只能清空水腔，不能改变基质自己的容器持水量。若基质过细，潮汐后即使抽回游离水，基质仍可能长时间高含水；若基质过粗，潮汐水位过低或时间过短时，主根区可能根本没有吸到足够水。

设计要求：

- 不要先写死 20 分钟为最终参数。
- 第一版实验应测不同配方的吸水曲线、排水后容器持水量和 24/48/72h 干回落曲线。
- 内盆底孔、颗粒级配、基质高度要标准化；产品如果允许用户随意换土，算法可靠性会显著下降。
- 对喜干植物，要单独测试高颗粒基质下“低水位 + 短浸泡 + 完全干透后再触发”的参数窗口。
- 对空气根/兰科/水苔/树皮介质不能套普通土培模型。

### 9.5 风险五：植物类型差异

同样的灌溉方式，对不同植物结果完全不同。

优先适配：

- 绿萝、吊兰、白掌、豆瓣绿、部分蕨类、菖蒲、网纹草等。

条件适配：

- 长寿花、蟹爪兰、玉露、仙人球、生石花等多肉类，不应套用观叶植物参数；只能在标准高颗粒基质、强光、低水位短浸泡、完全抽回、完全干透后再触发的条件下验证。
- 球根生长期可探索短时底灌或顶部维护浇水，但必须按物种和阶段管理；开花后衰退期要逐步减水。

不应直接适配：

- 水草、兰科附生、空气凤梨、食虫植物、球根休眠期、盆景木本长期室内展示。

因此，植物适配不应写成“能/不能”，而应写成三类规则：

| 规则类型 | 植物 | 产品含义 |
|---|---|---|
| 默认潮汐模式 | 中性/偏湿观叶和部分湿生小型植物 | MVP 主战场，可做“养不死”验证 |
| 条件潮汐模式 | 部分多肉、球根生长期、偏干但耐短时湿润植物 | 需要专用基质和参数，不作为首发承诺 |
| 停水/特殊模式 | 球根休眠期、食虫、兰科附生、水陆植物 | 不能套潮汐算法，需要单独 SKU 或养护规则 |

## 11. 推荐的 MVP 工程定义

### 11.1 花盆结构

建议使用双层结构：

```text
外盆：封闭水箱 + 泵 + 水位检测 + 可拆洗结构
内盆：有底孔/侧孔的种植篮，装标准化颗粒/半无机基质
两者之间：浅水浸泡区 + 回水路径
上部：开放植物空间 + 补光灯环 + 主动导风
```

不要做成：

```text
外盆永久封死
内盆底部长时间泡水
水箱无法清洗
只有注水没有抽回确认
```

### 11.2 灌溉流程

推荐基础流程：

```text
1. 判断是否需要浇水
   - 盆重下降 / 土壤湿度下降 / 距上次浇水天数 / 植物规则综合

2. 注水
   - 泵入底部浅水区
   - 控制到目标水位，而不是只按泵运行秒数

3. 浸泡
   - 初始实验：5 / 10 / 15 / 20 min 四档
   - 后续按吸水曲线选择最短有效时间

4. 抽回
   - 泵反转或独立回水泵
   - 抽到水位低于安全线

5. 记录
   - 注入量、抽回量、估算吸水量
   - 下次干回落时间

6. 通风
   - 抽回后运行风扇，降低土表和盆腔湿度
```

### 11.3 第一版参数建议

不要把“浸泡 20 分钟”直接写死成产品参数。建议这样做实验：

| 变量 | 实验范围 | 判断指标 |
|---|---|---|
| 水深 | 0.8 / 1.2 / 1.6 / 2.0 cm | 吸水量、底部残留、上层湿润情况 |
| 时间 | 5 / 10 / 15 / 20 min | 吸水量是否趋于平台 |
| 基质 | 赤玉土; 赤玉土+浮石; 赤玉土+椰糠少量; 颗粒土 | 毛细高度、干回落速度 |
| 植物 | 绿萝、白掌、豆瓣绿、菖蒲、长寿花、玉露 | 存活、根系、黄叶、徒长、烂根 |
| 风扇 | 无风 / 抽回后 10min / 抽回后 30min | 土表干燥、小黑飞、霉菌 |

## 12. 最低成本验证实验

### 实验 1：同基质上浇 vs 潮汐水分曲线

目的：验证“潮汐法是否真的更湿/更干”，避免继续停留在概念争论。

方法：

1. 同样内盆、同样基质、同样干燥初始重量。
2. A 组顶部浇透，直到盆底出水，静置并倒掉托盘水。
3. B 组底部潮汐，注入固定水深，浸泡到吸水平台，抽回水腔游离水。
4. 两组分别记录灌溉后 0h、1h、6h、24h、48h、72h 重量。
5. 如果有条件，在上层/中层/下层插入湿度探针，记录垂直含水分布。

判断：

- 如果两组最终重量接近，说明容器持水量主要由基质决定。
- 如果潮汐组下层长期更湿，要调整水位、时间、底孔和颗粒级配。
- 如果潮汐组上层长期过干但植物缺水，说明基质毛细能力不足。
- 这个实验决定后续是优化参数，还是更换基质。

### 实验 2：介质吸水曲线

目的：确定 20cm 花盆内，底部浸泡多久才足够。

方法：

1. 同样内盆、同样基质、同样干燥初始重量。
2. 底部注入固定水深。
3. 分别浸泡 5、10、15、20 分钟。
4. 抽回后称重。
5. 记录 24h、48h、72h 重量下降。

判断：如果 10 分钟和 20 分钟吸水量差异不大，20 分钟没有必要。

### 实验 3：底灌 vs 顶灌 vs 自吸盆

目的：验证方案是否真的让植物更稳。

组别：

| 组别 | 方式 |
|---|---|
| A | 人工顶部浇透，托盘倒水 |
| B | 自吸盆/棉绳连续供水 |
| C | 你的潮汐底灌：注水-浸泡-抽回 |
| D | 无自动浇水对照 |

植物：绿萝、白掌、豆瓣绿作为主验证；长寿花或玉露作为条件验证，不与观叶植物混用同一参数。

指标：

- 30 天存活率。
- 根系白根/烂根比例。
- 土表小黑飞/霉菌。
- 用户维护次数。
- 叶片状态和生长量。
- 盆重/湿度曲线。

### 实验 4：喜干植物低水位短潮汐分支

目的：验证“标准颗粒基质 + 低水位短潮汐”是否可以作为后续喜干模式，而不是凭直觉排除或承诺。

组别：

| 组别 | 方式 |
|---|---|
| A | 标准颗粒基质，顶部浇透，完全干透后再浇 |
| B | 标准颗粒基质，0.5 cm 低水位潮汐，5 min，完全抽回，完全干透后再触发 |
| C | 标准颗粒基质，1.0 cm 水位潮汐，10 min，完全抽回，完全干透后再触发 |

植物建议：长寿花、蟹爪兰、玉露、仙人球中选 2 类，不要一开始上生石花这类极窄窗口植物。

判断：

- B 组若存活率、根系状态接近 A 组，同时用户维护更少，可保留“喜干潮汐模式”作为后续 SKU/植物包。
- B 组若吸水不足、C 组若烂根上升，说明参数窗口太窄，不适合作为大众承诺。
- 球根只测生长期，不测休眠期潮汐；休眠期应验证停水提醒和状态机。

### 实验 5：断电/故障安全

目的：避免“系统坏了反而杀植物”。

测试：

- 注水后断电，水是否会长期泡根。
- 泵反转失败，是否有溢流/泄压/机械回退。
- 水位传感失败，是否会超量注水。
- 水箱满污水 7 天是否有异味、藻类、沉淀。

## 13. 最终建议

### 13.1 是否采用浸盆法

建议采用，但正式命名和设计上应坚持叫：

```text
浸盆式潮汐灌溉
```

不要叫：

```text
坐水浇灌
懒人自吸
持续底部供水
```

因为你真正要做的是 flood-and-drain / ebb-and-flow，而不是让植物一直坐在水里。

### 13.2 产品论证话术

建议内部使用：

> 植间采用短时底部潮汐供水，让标准化基质从下方吸水；随后系统抽回水腔中的游离水，并通过主动导风帮助基质回到透气状态。它不是长期坐水，而是可测量、可记录、可干回落的水分事件。

不要对外过早说：

> 适合所有植物、永不烂根、完全不用管、永远不用顶部浇水。

这些说法都没有证据，且会给后续产品埋雷。

### 13.3 MVP 必须保留的功能

| 功能 | 是否必须 | 理由 |
|---|---|---|
| 底部注水泵 | 必须 | 核心供水动作 |
| 抽回能力 | 必须 | 防止坐水和烂根 |
| 水位检测 | 必须 | 防注水过量、确认抽回 |
| 可拆洗水箱 | 必须 | 封闭系统卫生底线 |
| 主动导风 | 必须 | 与潮汐干湿循环形成闭环 |
| 周期冲盐提醒 | 必须 | 底灌天然缺陷补偿 |
| EC/pH 传感 | MVP 可选 | 若不自动投肥，可先不做；若做营养液则优先级上升 |
| 顶部自动冲洗 | MVP 可不做 | 结构复杂，可先用手动维护解决 |
| 标准化内盆/基质包 | 必须 | 潮汐算法依赖毛细吸水和容器持水量，不能让用户随意换土后仍承诺可靠 |

### 13.4 最关键的 PM 判断

浸盆式潮汐灌溉的价值不是“技术更酷”，而是它能把用户不会判断的浇水动作变成系统可控动作。

但是它仍然只是“让植物活得更久”的一部分。真正的产品假设是：

```text
补光 + 标准化基质 + 可控底灌 + 抽回游离水 + 主动导风 + 合适植物/植物阶段规则
  -> 比普通花盆显著降低死亡率
  -> 用户愿意为“植物不会轻易死”付费
```

下一步不要继续优化泵型号，先做土法实验验证这条链是否成立。

### 13.5 产品决策链是否通畅

当前逻辑链是通畅的，但必须按以下版本表达：

```text
用户痛点：想要桌面植物，但不会判断浇水、光照不足、容易养死
   -> 产品承诺：让植物更稳定地活着，而不是炫技
   -> 技术组合：补光解决低光，潮汐供水解决浇水动作不稳定，主动导风解决湿后干回落
   -> 工程边界：标准化内盆/基质，短时供水，抽回游离水，周期冲盐，水箱卫生
   -> 植物边界：首发中性/偏湿观叶；喜干植物只做验证分支；休眠/特殊植物单独状态机
   -> 验证方式：同基质水分曲线 + 植物 A/B 存活实验 + 用户支付意愿访谈
```

这条链最容易断的地方有三个：

1. 只强调“浸盆”，却没有控制基质，导致算法在用户随意换土后失效。
2. 只验证泵能注水/抽水，却没有验证 24/48/72h 干回落和根系状态。
3. 对外说“适合所有植物”，但实际不同植物需要不同基质和生理阶段规则。

只要把这三个点管住，浸盆式潮汐灌溉作为 MVP 主方案是合理的。
