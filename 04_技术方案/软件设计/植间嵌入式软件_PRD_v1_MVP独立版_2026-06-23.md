## 1. 产品概述

### 1.1 软件定义

植间嵌入式软件运行在 ESP32-S3 主控上，负责执行器控制、传感器采集、摄像头拍照、音频采集、IMU 敲击检测、本地安全保护、状态机管理、事件日志记录，并通过局域网 API 与 APP 通信。

一句话定义：

> **植间嵌入式软件是硬件安全执行层和本地设备服务端。**

### 1.2 核心目标

MVP 阶段嵌入式软件需要完成：

1. 连接 Wi-Fi，并提供局域网 IP 访问；
2. 向 APP 暴露 HTTP/WebSocket 接口；
3. 接收 APP 控制请求；
4. 安全控制注水、排水、补光灯、风扇、雾化器；
5. 使用本地定时器完成超时自动关断；
6. 采集单电容土壤湿度；
7. 摄像头每小时拍照，并支持 APP 请求立即拍照；
8. 采集麦克风音频流或生成音频事件；
9. 识别陀螺仪敲击/移动事件；
10. 记录动作、传感器、照片、音频、运动事件日志；
11. 在 APP 断连时仍保证执行器不会无限运行。

### 1.3 嵌入式软件的职责

| 职责 | 说明 |
|---|---|
| 硬件控制 | 控制泵、灯、风扇、雾化器 |
| 安全保护 | 超时关断、互斥保护、参数校验、all_stop |
| 传感采集 | 单电容湿度、摄像头、麦克风、IMU |
| 状态机 | 管理执行器运行状态和完成原因 |
| 网络服务 | 提供 HTTP/WebSocket API |
| 本地日志 | 记录动作事件、传感事件、错误事件 |
| 媒体服务 | 保存/提供最新照片和历史照片 |
| 时间同步 | 维护本地时间，用于日志和拍照任务 |

### 1.4 嵌入式软件不负责

| 不负责 | 说明 |
|---|---|
| APP UI | 嵌入式只提供接口和状态 |
| 植物档案搜索 | 由 APP/AI 服务完成 |
| 长文本日记生成 | 嵌入式只提供数据和事件 |
| 用户账号 | 不在设备端实现 |
| 商业化订阅 | 不在 MVP 嵌入式范围内 |
| 复杂植物养护策略 | MVP 以基础控制和安全保护为主 |

## 5. 执行器状态机

### 5.1 通用状态

每个执行器都应至少支持以下状态：

```text
idle
→ starting
→ running
→ stopping
→ completed / timeout_stopped / stopped_by_app / failed / rejected
```

### 5.2 指令通用字段

APP 下发控制请求时，嵌入式应识别：

| 字段 | 说明 |
|---|---|
| command_id | APP 生成的指令 ID |
| type | 指令类型 |
| requested_duration_ms | 请求运行时长 |
| source | app_manual / app_pomodoro / scheduler |
| created_at | APP 创建时间，可选 |

嵌入式返回：

| 字段 | 说明 |
|---|---|
| accepted | 是否接受 |
| state | 当前状态 |
| max_duration_ms | 嵌入式允许的最大时长 |
| actual_duration_ms | 实际采用时长 |
| reason | 拒绝或完成原因 |

### 5.3 完成原因

| 原因 | 说明 |
|---|---|
| completed | 正常完成 |
| stopped_by_app | APP 提前停止 |
| timeout_stopped | 本地安全定时器停止 |
| rejected_interlock | 互斥保护拒绝 |
| rejected_param | 参数越界拒绝 |
| rejected_busy | 设备忙 |
| failed_hardware | 硬件执行失败 |
| all_stopped | all_stop 触发 |

## 9. 互斥与安全规则

### 9.1 互斥规则

| 动作 A | 动作 B | 规则 |
|---|---|---|
| 注水 | 排水 | 不允许同时运行 |
| 注水 | all_stop | all_stop 优先 |
| 排水 | all_stop | all_stop 优先 |
| 雾化 | all_stop | all_stop 优先 |
| 番茄钟呼吸 | light_off | 手动关灯优先 |

### 9.2 超时规则

| 执行器 | 必须超时关断 |
|---|---|
| 注水 | 是 |
| 排水 | 是 |
| 风扇 | 是 |
| 雾化器 | 是 |
| 补光灯呼吸 | 是 |

### 9.3 断连规则

| 场景 | 嵌入式行为 |
|---|---|
| APP 断连，注水运行中 | 按本地定时器自动停止 |
| APP 断连，排水运行中 | 按本地定时器自动停止 |
| APP 断连，雾化运行中 | 按本地定时器自动停止 |
| Wi-Fi 断开 | 不影响本地安全关断 |
| APP 重连 | 通过 `/api/status` 返回真实状态 |

## 12. 验收标准

### 12.1 控制验收

| 项目 | 标准 |
|---|---|
| 注水 | start 后运行，超时自动停止，可被 APP 提前停止 |
| 排水 | start 后运行，超时自动停止，可被 APP 提前停止 |
| 注水排水互斥 | 注水中排水请求被拒绝，或按规则处理 |
| 补光灯 | 可开关，可执行番茄钟呼吸并恢复状态 |
| 风扇 | 可开关，超时自动停止 |
| 雾化器 | 可开关，超时自动停止 |
| all_stop | 运行中执行器全部停止 |

### 12.2 采集验收

| 项目 | 标准 |
|---|---|
| 湿度 | 可读取 raw 和等级 |
| 拍照 | 连续 24 小时至少成功 20 张 |
| 最新图像 | `/api/photos/latest` 可获取 |
| 请求拍照 | APP 请求后可生成新照片 |
| 音频流 | 可启动、停止、查询状态 |
| 音频事件 | 可输出音频事件或摘要 |
| 敲击 | 敲击事件可被识别并记录 |

### 12.3 网络验收

| 项目 | 标准 |
|---|---|
| IP 连接 | APP 可通过 IP 连接设备 |
| 状态接口 | `/api/status` 返回完整状态 |
| 控制接口 | `/api/commands` 可接收并返回结果 |
| 事件接口 | `/api/events` 可返回日志 |
| 断连保护 | APP 断连后执行器不会无限运行 |

## 13. 风险与约束

| 风险 | 应对 |
|---|---|
| Wi-Fi 长连接不稳定 | 控制使用 HTTP 短请求，事件流可重连 |
| 摄像头发热 | 拍照频率可配置，避免持续视频采集 |
| 音频隐私 | 不默认长期保存原始音频 |
| 存储不足 | 照片压缩，保存周期可配置 |
| 执行器误动作 | 本地超时、互斥、all_stop |
| APP 状态不同步 | 重连后以 `/api/status` 为准 |
| 电容湿度漂移 | 提供 raw 和等级，APP 不展示伪精确值 |

## 15. 最终说明

植间嵌入式软件 MVP 的核心不是复杂自动养护算法，而是可靠地完成本地硬件控制、采集和安全保护。

APP 可以随时请求动作或提前停止动作，但嵌入式必须独立保证：

> **任何执行器都不会因为 APP 断连、网络失败或用户忘记停止而无限运行。**

只有嵌入式安全闭环成立，APP 的控制台、照片记录、音频日志、敲击互动和 AI 日记才有可靠基础。
