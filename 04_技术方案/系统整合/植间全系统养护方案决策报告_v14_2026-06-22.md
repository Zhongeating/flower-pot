# 植间全系统养护方案决策报告

> 日期：2026-06-22  
> 用途：理解方案、产品汇报、样机验证前评审  
> 对象：20 cm 桌面智能花盆，混配基质包、多高度电容检测、低水位潮汐供水、低维护养护系统

---

## 0. 一页结论

### 0.1 推荐方案

植间采用四层养护系统。

| 层级 | 模块 | 主要职责 |
|---|---|---|
| 基质层 | 混配基质包、内盆结构、可选铺面/苔藓层 | 定义根区的水分、空气、结构和化学环境 |
| 执行器层 | 低水位潮汐底灌、微风扇、超声波雾化、LED 灯环 | 执行供水、通风、加湿和补光 |
| 感知层 | 多高度电容检测、摄像头 + LLM | 记录基质响应曲线，识别植物阶段和异常 |
| 维护层 | 缓释肥棒、高水位冲盐、换水提醒、手动喷瓶 | 处理低频施肥、冲盐、换水和病虫害 |

主供水建议采用低水位潮汐底灌。它保留混配基质作为根区缓冲层，能在泵、传感器、用户维护和环境波动出现偏差时给植物留出恢复空间。

### 0.2 当前关键判断

| 问题 | 判断 |
|---|---|
| 主供水方案 | 低水位潮汐底灌 |
| 核心理由 | 桌面干净、土表相对干、供水可主动结束、基质有缓冲 |
| 混配基质 | 必须作为产品组件管理，不能依赖用户随意换土 |
| 多高度电容检测 | 值得做。它能让每次潮汐浸盆成为可观测的水分响应测试 |
| 方案边界 | 先验证观叶和部分湿生花卉；特殊植物做实验包 |

### 0.3 样机起测参数

| 动作 | 初始水位 | 初始时长 | 用途 |
|---|---|---|---|
| 日常潮汐供水 | 有效基质高度 1/4-1/3 | 5-10 min | 保守供水，观察 H2/H3 响应 |
| 干透救水 | 1/3-1/2 | 10-20 min | 重新润湿过干或疏水基质 |
| 高水位冲盐 | 约 2/3 起测 | 10-15 min | 验证溶盐与抽回效果 |
| 边界测试 | 2/3-3/4 | 按实验调整 | 观察上浮、溢水和安全边界 |

这些参数是样机起点。最终参数由基质水分曲线、EC 曲线、植物状态和安全边界决定。

## 1. 设计目标

植间要降低桌面植物长期养护难度。用户不需要每天判断浇水，也不需要掌握冲盐、肥料、光照、休眠期和基质物理性质。

| 目标 | 产品要求 | 对应模块 |
|---|---|---|
| 长期存活 | 稳定供水、补光、湿度和通风 | 潮汐底灌、灯环、雾化、风扇 |
| 桌面干净 | 封闭水路，减少托盘积水和溅水 | 外盆水腔、泵抽回、内盆防漏 |
| 少虫少霉 | 土表不长期潮湿，局部空气有交换 | 底部供水、微风扇、阶段雾化 |
| 低维护 | 维护动作低频、简单、可提醒 | 缓释肥棒、换水提醒、手动喷瓶 |
| 可解释 | 用户知道系统当前策略 | App 状态、摄像头识别、异常提醒 |
| 可学习 | 每次潮汐产生数据 | 多高度电容检测、响应曲线分析 |

## 2. 系统架构

### 2.1 基质层

基质层决定根区环境。它影响持水、通气、毛细上升、盐分迁移和传感器读数。植间应把基质作为产品组件管理。

| 内容 | 作用 |
|---|---|
| 混配基质包 | 控制水分和空气比例 |
| 内盆种植篮 | 控制水进入基质和排出基质的路径 |
| 防漏网 | 防止颗粒进入水腔和泵路 |
| 可选铺面 | 美观、减少飞溅、辅助观察干湿 |
| 苔藓附加层 | 增加微景观效果，需独立湿度和低肥策略 |

### 2.2 执行器层

| 模块 | 功能 | 注意点 |
|---|---|---|
| 低水位潮汐底灌 | 根区供水 | 参数按基质校准 |
| 微风扇 | 轻微空气交换，减少表面积水 | 不宣传强力催干 |
| 超声波雾化 | 局部湿度管理 | 花期、低温弱光、旱生植物降档 |
| LED 灯环 | 桌面补光 | 需要实测 PPFD |

### 2.3 感知层

| 模块 | 功能 | 注意点 |
|---|---|---|
| 多高度电容检测 | 记录不同高度的水分响应 | 读数按基质包校准 |
| 水位信号 | 判断注水、抽回和异常 | 可先复用电容信号，保留冗余位置 |
| 摄像头 + LLM | 识别花期、休眠、发芽、黄叶、病斑 | 高风险维护只提醒用户确认 |

### 2.4 维护层

| 模块 | 功能 | 注意点 |
|---|---|---|
| 缓释肥棒 | 低频施肥 | 降低盐峰值，不代表无盐 |
| 高水位冲盐 | 尝试自动化溶盐、排盐 | 需要验证效率，保留顶部淋洗备用 |
| 换水提醒 | 移除水箱旧水 | MVP 可先用定时提醒 |
| 手动喷瓶 | 用户处理虫害和病害 | 不做自动喷药 |

## 3. 主供水与栽培方式横向对比

| 方案 | 方法 | 优点 | 桌面花盆限制 | 结论 |
|---|---|---|---|---|
| 顶部浇透 | 从上方浇至底部排水 | 润湿确定，冲盐可靠 | 漏水、土表湿、依赖用户 | 备用维护动作 |
| 自吸盆/SIP | 储水层通过棉绳或介质连续供水 | 结构简单、安静 | 主动控制弱，干湿周期不清晰 | 不作主方案 |
| 毛细垫 | 盆底接触湿垫吸水 | 适合多盆统一管理 | 单盆桌面清洁和美观较差 | 不作主方案 |
| 滴灌 | 滴头或管路供水 | 农业成熟，水肥一体方便 | 管路、滴头、堵塞、局部湿表 | 单盆不优先 |
| DWC 深水培 | 根系浸入充氧营养液 | 结构清晰，叶菜常见 | 水质、缺氧、换液和 EC/pH 管理 | 专项植物包 |
| NFT 营养液膜 | 薄层营养液流过根系 | 适合叶菜和管道系统 | 桌面单盆结构复杂，断流风险 | 不作主方案 |
| 半水培/LECA | 惰性颗粒 + 储水层 | 干净，可视化强 | 植物换根和营养液管理复杂 | 专项植物包 |
| 气雾培/空气培 | 根系悬空，由细雾供水供肥 | 根区氧气高，水肥效率高，根系可视化强 | 喷雾、泵、电力、喷头清洁和营养液标准化要求高，故障缓冲小 | 专项观察 |
| Kratky 被动水培 | 营养液静置，水位下降形成空气根区 | 极简、无泵，适合短周期叶菜 | 长期观赏植物会遇到营养液、水位和换液问题 | 专项实验 |
| 鱼菜共生/bioponics | 微生物把有机营养转化为植物可吸收养分 | 生态叙事强，可持续价值高 | 水质、微生物、气味、标准化和维护复杂 | 远期概念 |
| 低水位潮汐底灌 | 底部短时注水，基质吸水后抽回 | 水路封闭、土表相对干、可主动结束、有基质缓冲 | 不天然冲盐，参数依赖基质 | 当前主供水优选 |

## 4. 混配基质系统

### 4.1 专业依据

Purdue Extension、Rutgers NJAES、Arkansas Extension 等资料都把容器基质看作由多个组件构成的系统。不同组件组合后，会改变总孔隙率、容器持水量、空气容量、容重、pH、EC 和结构稳定性。

| 专业观点 | 对植间的意义 |
|---|---|
| 物理性质对容器基质很关键 | 装盆后难调整，基质包阶段要设计好 |
| 单一材料常有短板 | 需要组合保水、通气和结构组分 |
| 比例服务于作物和容器 | 不能套用一个万能配方 |
| 基质要匹配灌溉方式 | 潮汐底灌需要关注毛细上升和排水曲线 |

### 4.2 组件横向对比

| 组件 | 主要贡献 | 主要风险 | 适合角色 |
|---|---|---|---|
| 泥炭 | 保水、保肥、轻 | 干透后疏水，过量偏湿，资源争议 | 保水主体 |
| 椰糠 | 保水、较易复水、结构稳定 | 来源差异大，可能 EC/钠/氯偏高 | 泥炭替代或保水主体 |
| 松树皮/树皮块 | 通气、结构、抗压 | 保水较低，分解后变细 | 结构骨架 |
| 赤玉土 | 保水和通气较平衡，颗粒可视 | 长期粉化 | 标准化矿物主体 |
| 鹿沼土 | 酸性、轻、保水 | 过轻，适配面有限 | 喜酸植物组分 |
| 浮石/火山石 | 稳定、通气、排水 | 毛细上升有限 | 通气和结构组分 |
| 珍珠岩 | 轻、通气、常见 | 易上浮，粉尘，保水有限 | 通气组分 |
| 蛭石 | 保水、保肥 | 过量易湿，长期结构下降 | 育苗或高保水少量组分 |
| 沙/细砂 | 增重、增加接触 | 过细会降低空气孔隙 | 调整颗粒级配 |
| 陶粒/LECA | 稳定，适合半水培 | 保肥弱，毛细能力需验证 | 半水培实验组分 |

### 4.3 基质包开发流程

| 步骤 | 输出 |
|---|---|
| 选目标植物 | 明确湿生、旱生、喜酸、附生或苔藓 |
| 设物理目标 | 持水、空气容量、毛细上升、容重、EC/pH |
| 选组件 | 保水主体 + 结构骨架 + 通气材料 + 化学调节 |
| 做比例小样 | 至少 3 个比例梯度 |
| 测水分曲线 | 潮汐后 H1-H4 响应、排水、干燥 |
| 测 EC/pH | 冲盐前后、缓释肥释放曲线 |
| 种植验证 | 2-8 周观察根系、叶片和视觉状态 |

## 5. 花盆结构与水腔

Illinois Extension 指出，普通花盆底部铺 gravel layer 不会改善排水，水会在细基质和粗颗粒层交界处滞留。植间的底部水腔属于可控制工程结构，价值来自注水、抽回和检测。

推荐剖面：

```text
植物冠层
可选铺面 / 苔藓附加层
标准化混配基质包
防漏网与内盆透水结构
外盆潮汐水腔
泵路、水位检测、防溢流结构
```

## 6. 多高度电容检测

### 6.1 核心价值

每次潮汐浸盆都是一次带已知输入的系统测试。

| 输入 | 输出 |
|---|---|
| 水位、注水量、浸泡时间、抽水时间 | H1-H4 电容响应曲线 |
| 灯光、风扇、环境温湿度 | 排水曲线、干燥曲线 |
| 植物阶段 | 下一轮水位、时长、间隔 |

### 6.2 可估算指标

| 指标 | 产品意义 |
|---|---|
| 吸水启动时间 | 底部供水是否成功 |
| 毛细上升延迟 | 水能否到达目标根区 |
| 最大湿润高度 | 当前水位是否足够 |
| 排水半衰期 | 底部是否滞水 |
| 干燥斜率 | 植物耗水和环境变化 |
| 层间梯度 | 底湿上干、全层偏湿或上层过干 |

### 6.3 建议高度

| 层位 | 位置 | 用途 |
|---|---|---|
| H1 | 底部 1-2 cm / 水腔上方 | 注水到位、抽回完成 |
| H2 | 有效高度 1/3 | 日常低水位是否有效 |
| H3 | 有效高度 1/2 | 干透救水和标准湿润 |
| H4 | 有效高度 2/3 | 高水位冲盐和上层湿润边界 |

HortScience 2015、SciELO salvia、HortTechnology 2013 和 Sensors 2025 等资料支持电容传感器用于底灌控制和基质水分监测，也强调基质特异性校准。

## 8. 验证计划

### 8.1 P0 实验

| 实验 | 目的 | 通过标准 |
|---|---|---|
| 基质混配小样 | 比较 3-5 个比例 | 毛细上升、空气容量、干燥曲线可区分 |
| 多高度电容响应 | 验证 H1-H4 曲线 | 各层响应有稳定时序 |
| 低水位潮汐 | 确认日常供水参数 | H2/H3 响应，上层不过湿 |
| 基质上浮/溢水 | 验证高水位安全 | 无明显漂浮、溢水、堵塞 |
| 灯环 PPFD | 确认植物支持范围 | 低/中光目标达成 |

### 8.2 P1 / P2 实验

| 实验 | 目的 |
|---|---|
| 高水位冲盐效率 | 判断能否带走盐分 |
| 缓释肥 EC 曲线 | 确定冲盐周期 |
| 微风扇表面湿度 | 判断风扇收益 |
| 摄像头阶段识别 | 判断自动策略准确率 |
| 气雾培概念观察 | 只用于根系可视化和专项包探索 |
| Kratky 被动水培小样 | 评估叶菜/香草短周期专项包可能性 |

## 9. 汇报建议

### 9.1 推荐表达

> 植间选择低水位潮汐底灌作为首代主线，是因为混配基质能提供水分和养分缓冲，系统故障和用户维护偏差不会立刻伤害根系。桌面长期观赏植物更需要稳定、可校准、容错率高的根区环境。

### 9.2 避免表达

| 表达 | 风险 |
|---|---|
| 气雾培一定优于潮汐底灌 | 气雾培效率高，但故障缓冲低、营养液标准化复杂 |
| Kratky 零维护 | 适合短周期叶菜，长期观赏植物仍有营养液和水位问题 |
| 一个万能土配方支持所有植物 | 植物和灌溉方式差异太大 |
| 盆底铺陶粒能改善排水 | 专业资料认为这常常会抬高悬挂水层 |
| 多高度电容直接测导水率 | 应写成估算吸水、排水和干燥特征 |
