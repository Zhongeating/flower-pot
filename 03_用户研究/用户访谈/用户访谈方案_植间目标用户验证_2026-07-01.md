## 零、核心访谈原则

不是来推销产品，而是来验证：**这个人是否真的需要一盆真实、会变化、低精力的桌面植物。**

### 6 大原则

| 原则 | 具体做法 |
|------|----------|
| 先问过去，再问未来 | 问"你之前养过什么"，不问"你会不会买智能花盆" |
| 先问场景，不先讲功能 | 问"你桌面现在怎么样"，不马上讲补光和底灌 |
| 先让用户自由讲，再追问细节 | 用开放题，然后追问"具体是什么时候""你觉得最麻烦的是什么" |
| 问行为，不问态度 | 问"你买过什么""你花过多少钱"，不问"你喜不喜欢" |
| 用对比迫出优先级 | 问"你更在意省心还是开花"，而不是"你在意开花吗"（用户会说都在意）|
| 看留资行为，不看口头许诺 | "听起来不错"≠真需求。愿意报名试用 / 交意向金才是信号 |

## 一、需要验证的核心假设

### 7 层假设（优先级从高到低）

| 假设 | 验证问题 | 当前状态 | 风险等级 |
|------|----------|----------|----------|
| H1 目标用户存在 | 存在一批固定办公/学习桌长期停留、养死过桌面植物、仍想养开花真植物、已为桌面/植物付费、不想高频照料的人 | 待验证 | 🔴 致命 |
| H2 场景真实 | 用户确实想让植物留在自己最常待的桌面，而不是窗台、阳台、客厅 | 待验证 | 🔴 致命 |
| H3 痛点真实 | 弱光、养护判断、卫生风险、维护打断、精力消耗真的阻碍用户继续养桌面植物 | 部分验证（竞品评论） | 🔴 高 |
| H4 成长价值成立 | 用户不只想要绿色装饰，还在意新叶、花苞、开花、时间线等可见成长 | 待验证 | 🔴 高 |
| H5 低精力约束成立 | 用户希望省心、低频维护、少判断、少打扰，而不是愿意每天认真养护 | 待验证 | 🔴 高 |
| H6 方案接受度成立 | 用户能理解"补光 + 底灌 + 基质 + 传感"不是伪智能，而是在帮植物活好 | 待验证 | 🟡 中 |
| H7 付费意愿成立 | 用户愿意为低精力桌面开花真植物支付覆盖成本的价格 | 待验证 | 🟡 中 |

**优先验证顺序**：H1 → H2 → H3 → H4 → H5 → H6 → H7

**如果哪一层不成立的处理**：

- H1 不成立 → 重新定义目标用户
- H2 不成立 → 产品改为家居/阳台路线
- H3 不成立 → 重新评估保活系统价值
- H4 不成立 → 产品退化为低维护绿植
- H5 不成立 → 重新设计维护流程和交互
- H6 不成立 → 重新做方案表达和概念图
- H7 不成立 → 重新评估 BOM 和定价

## 二、访谈分层框架

访谈分 7 层，每层 1 个目标，递进验证。**总时长 45-60 分钟。**

```text
第 0 层（3-5 分钟）  ← 筛选是否是目标用户
     ↓
第 1 层（5-7 分钟）  ← 还原桌面场景和停留时长
     ↓
第 2 层（8-10 分钟） ← 挖植物失败经历和原因
     ↓
第 3 层（5-7 分钟）  ← 判断替代方案是否满足
     ↓
第 4 层（10-12 分钟）← 挖成长价值和低精力约束
     ↓
第 5 层（8-10 分钟）← 展示概念，测试理解和吸引点
     ↓
第 6 层（8-10 分钟）← 测试价格和行动意愿
```

## 三、第 0 层：筛选目标用户

**目标**：快速判断是不是值得深访的人。

**时长**：3-5 分钟

**问题和判断标准**

| 维度 | 问法 | 筛选标准 |
|------|------|----------|
| 固定桌面 | 你每天在固定办公桌/书桌前大概多久？这个桌面是长期使用的吗？ | 需要≥4小时/天，且是长期停留位置 |
| 桌面付费 | 过去一年你有没有买过桌面绿植、摆件、灯、支架、键鼠、香薰等桌面物品？大概花了多少钱？ | 需要有至少一次¥50以上的购买 |
| 植物经历 | 你有没有在桌面或工位养过植物？养过哪些？现在还在吗？ | 必须有植物养护经历 |
| 失败经历 | 有没有养死、养坏、不开花、黄叶、搬走的经历？ | 必须有失败或妥协经历 |
| 重试意愿 | 如果之前失败过，你现在还想再养一盆吗？想养什么样的？ | 必须明确表示想再养 |
| 低精力 | 你愿意每天花时间照料，还是更希望它不要增加太多待办？ | 必须表示不想高频照料 |

**进入深访门槛**：上述 6 个维度**全部达标**为"高匹配用户"。

如果任何维度不达标，可记录为"可观察用户"，继续访谈作为对照。

## 四、第 1 层：还原桌面场景

**目标**：确认"桌面"是不是高价值场景，用户是否真的经营和在意桌面环境。

**时长**：5-7 分钟

### 正面问法

```
Q1：你现在的办公桌/书桌上通常放什么？
→ 听用户罗列。可能回答：电脑、显示器、键盘、鼠标、书、本子、笔、手机、灯、摆件、植物等。

Q2：你有没有刻意布置过桌面？为什么想布置？
→ 看用户是否主动经营桌面。如果只是堆放工作必需品，匹配度低。

Q3：桌面上有没有让你放松、分心、陪伴、提醒自己的东西？
→ 关键词：放松、陪伴、安定感。看用户是否投入情绪价值。

Q4：你买桌面物品时，更看重好看、实用、情绪价值，还是别的？
→ 判断用户的优先级顺序。
```

### 侧面问法（如果回答不充分）

```
Q5：你桌面上有没有什么东西是别人觉得没必要，但你自己挺在意的？
→ 听用户讲那些别人看不懂的执念。

Q6：你有没有因为桌面太乱/太冷/太工具化而想调整它？
→ 看用户是否曾为桌面环境感到困扰。

Q7：你有没有收藏过小红书/淘宝/抖音里的桌搭内容？
→ 看用户是否对桌搭文化有兴趣。
```

### 关键追问

```
Q8：这个东西你用了多久？有没有新鲜感过去？
→ 判断用户对桌面投资的持久性。

Q9：你为什么没有把它收起来？
→ 看用户是否认为这东西有长期价值。

Q10：如果桌面只能保留 3 个非工作必需品，你会留什么？
→ 用优先级逼出用户最在意的是什么。
```

### 判断标准

| 信号 | 含义 |
|------|------|
| 用户桌面完全工具化，不愿为体验花钱 | 低匹配 |
| 用户提到至少 2 个非工作必需品，且说有感情价值 | 中等匹配 |
| 用户主动讲过布置/调整桌面，提过"放松""陪伴" | 高匹配 |
| 用户提到小红书/抖音桌搭或曾为桌面花过¥300+ | 很高匹配 |

## 五、第 2 层：挖植物失败经历

**目标**：确认痛点是不是来自真实经历，而不是想象。

**时长**：8-10 分钟

### 正面问法（还原具体经历）

```
Q1：你上一次养桌面植物是什么时候？
→ 越近越好。如果是 1-2 年前，用户可能遗忘细节。

Q2：当时为什么买它？谁建议你的，还是自己想要？
→ 判断是否是主动需求。

Q3：它是什么植物？（绿萝、绣球、长寿花等）
→ 了解植物品类。开花类 > 绿叶类（成长价值更强）。

Q4：放在哪里？离窗户远吗？你觉得那个位置光线怎么样？
→ 判断是否是弱光场景。

Q5：每天你怎么判断要不要浇水？怎么浇的？
→ 听用户讲养护判断过程。观察是否是"猜""查教程""靠运气"。

Q6：最后它怎么样了？（黄叶、枯死、徒长、不开花、搬走）
→ 明确失败模式。

Q7：你觉得它失败/变差的最大原因是什么？
→ 关键问题。听用户自己说。不要引导。可能原因：浇水不对、光不够、长虫、搬动太多。
```

### 侧面问法（如果用户不够具体）

```
Q8：你有没有因为不知道怎么养而查过教程？
→ 看用户投入了多少认知成本。

Q9：有没有买过营养液、补光灯、自动浇水器、懒人盆？花过多少钱？
→ 用户曾经尝试过哪些解决方案。

Q10：植物状态变差时，你第一反应是什么？
→ 追问行为链：查原因 / 换位置 / 放弃 / 再买一盆？

Q11：有没有想过在教程上再花时间、学怎么养？
→ 判断用户是否愿意投入学习成本。
```

### 关键追问

```
Q12：植物死了/搬走了以后，你会不会有点挫败、内疚，还是觉得无所谓？
→ 最关键问题。挫败/内疚 = 用户有生命附着。无所谓 = 低匹配。

Q13：如果当时有东西能帮你解决一个问题，你最希望解决哪一个？
→ 优先级逼问。听用户说最关键的痛点。

Q14：之后的日子里，你还会想起它吗？现在想起来是什么感受？
→ 判断用户对这件事的心理影响深度。
```

### 判断标准

| 信号 | 含义 |
|------|------|
| 用户觉得植物死了无所谓、下次再买 | 装饰型，低生命附着 |
| 用户能具体说出失败原因（光照、浇水判断等） | 痛点真实 |
| 用户说起失败经历时有一点遗憾/内疚 | 有生命附着，会想再试 |
| 用户失败原因集中在弱光、浇水判断、维护麻烦 | 痛点吻合植间方案 |

## 六、第 3 层：替代方案满足度

**目标**：确认"为什么不是普通绿植/假花/电子桌宠"，判断替代方案缺口。

**时长**：5-7 分钟

### 逐个问替代方案

```
Q1：绿萝、豆瓣绿、白掌这类易养绿植能满足你吗？为什么？
A 满足 → 用户可能只需要好看 + 容易活，低需求。
B 不满足，因为没变化/一直不开花 → 用户在意成长。
C 不满足，因为养不活 → 用户重视保活。

Q2：你有没有想过养会开花的植物？为什么没养/没养好？
A 没想过 → 成长价值不强。
B 想过但试过失败了 → 正好是痛点。
C 想过但觉得麻烦 → 可能是低精力需求。

Q3：如果只是好看，假花或摆件可以吗？差在哪里？
A 可以 → 用户不需要真生命感。
B 不可以，感觉不真实/缺少生命感 → 用户在意真实性。
C 可以陪一段时间，但会腻 → 用户需要持续变化。

Q4：电子桌宠/屏保绿植会让你有陪伴感吗？会不会觉得不真实？
A 有陪伴感 → 用户对形式不太在意。
B 有点，但不真实 → 用户偏好真实植物。
C 没有，不真实/容易无感 → 用户看重真实和变化。

Q5：鱼缸、微景观生命感更强，你会考虑吗？担心什么？
A 会考虑 → 用户对生态感兴趣。
B 不会，太复杂/太容易出问题 → 用户是低精力型。
C 不会，不合适放桌上 → 用户认识到桌面限制。

Q6：你见过 Ivy 这类智能花盆吗？第一反应是想买、可爱、还是智商税？
A 想买 → 用户对智能花盆有期待。
B 可爱但不实用 → 用户看重形式。
C 智商税/伪智能/不可靠 → 用户对品类信任度低。
```

### 关键追问

```
Q7：如果你只能在"假花永远好看"和"真植物可能死但会长"之间选，你选哪个？
→ 核心问题，逼出用户的真实优先级。

Q8：如果一盆绿植能活但几乎没变化，你会不会很快无感？
→ 判断成长价值是否必要。

Q9：你更想要"稳定好看"，还是"真的会慢慢变好"？
→ 再次确认用户的核心需求是什么。
```

### 判断标准

| 信号 | 含义 |
|------|------|
| 用户接受假花/普通绿植，觉得差不多 | 需求偏装饰，低成长需求 |
| 用户明确说不一样、想要真植物 | 有真实性需求 |
| 用户说想看到变化、新叶/开花很关键 | 成长价值很强 |
| 用户对 Ivy 的评价是可爱但不实用/伪智能 | 对品类失望，需要重塑信任 |

## 七、第 4 层：成长价值和低精力约束

**目标**：这是最关键的一层。同时验证 H4（成长价值）和 H5（低精力约束）。

**时长**：10-12 分钟

### 成长价值探挖

#### 正面问法

```
Q1：你会在意植物长新叶、冒花苞、开花吗？
→ 听用户的真实反应。不是问"喜不喜欢"，而是问"在意吗"。

Q2：如果一盆植物只是活着但不怎么变（比如活一年还是那样），你还会持续关注吗？
→ 判断植物的持续吸引力。

Q3：你有没有拍过植物变化的照片？为什么拍？拍了以后做什么？
→ 听用户讲行为。可能回答：分享、记录、看自己养的成果等。

Q4：你看到自己养的植物开花，会有什么感觉？
→ 关键问题。听用户讲情绪。可能回答：成就感、惊喜、欣慰等。
```

#### 侧面问法

```
Q5：你有没有养成过什么长期小东西？（比如鱼、宠物、手账、打卡、收藏）
→ 了解用户是否有长期追踪小东西的习惯。

Q6：你会不会喜欢那种"过几天看一眼，它又变了一点"的东西？
→ 判断用户对"变化"本身的喜好。

Q7：如果植物每周自动生成一张成长对比图（新叶变多了、花苞冒出来了），你会不会看？会不会分享？
→ 测试时间线形式的吸引力。
```

#### 关键追问

```
Q8：你说起植物开花时眼睛亮不亮？
→ 观察用户的非语言反应。

Q9：如果可以记录植物从小到大的过程，你会不会觉得有意义？
→ 测试记录价值。
```

### 低精力约束探挖

#### 正面问法

```
Q10：你愿意多久照料一次植物？每天、每周、想起来再说？
→ 关键问题。听用户说自己的舒适频率。

Q11：你最不想做的养护动作是什么？（浇水、判断光照、清理水渍、查教程、换盆等）
→ 让用户列举最讨厌的任务。

Q12：如果植物需要频繁提醒你（比如每天通知一次"今天需要浇水"），你会觉得有帮助还是打扰？
→ 测试提醒频率的接受度。

Q13：你平时收到多少手机通知？你对哪些通知感到厌烦？
→ 判断用户的通知疲劳度。
```

#### 侧面问法

```
Q14：你现在生活/工作中有哪些提醒已经让你烦？
→ 听用户讲打扰来源。

Q15：你喜欢设备主动通知，还是只有出问题时通知？
→ 判断用户的交互习惯。

Q16：如果一个东西平时不说话，只在缺水/需要处理时提醒，你能接受吗？
→ 测试"克制交互"的可接受度。
```

#### 关键追问

```
Q17：你说"省心"，具体是什么意思？
→ 让用户定义"省心"。

Q18：省心到什么程度你才会觉得值得？
→ 判断最低可接受的维护频率。

Q19：如果每 7-10 天加一次水、偶尔清洁水箱，你能接受吗？还是觉得太麻烦？
→ 测试具体的维护频率。
```

### 判断标准

| 成长价值信号 | 含义 |
|---|---|
| 用户觉得只活着就够了 | 低成长需求 |
| 用户说"会有点在意变化" | 中等成长需求 |
| 用户说"想看开花""会记录"且眼睛亮 | 高成长需求 ✓ |

| 低精力信号 | 含义 |
|---|---|
| 用户愿意每天认真照料 | 不是目标用户 |
| 用户不确定、"都可以" | 动机不强 |
| 用户明确说想省心、不想高频维护且能举例说"我怕……" | 低精力约束成立 ✓ |

## 八、第 5 层：展示概念，测试理解

**目标**：在用户自然表达的基础上，展示植间方案，测试是否被理解、是否有吸引力。

**时长**：8-10 分钟

### 概念展示顺序

不要一开始展示完整炫酷产品图。否则用户会顺着你说好看。

**展示顺序**：

1. 场景图（桌面环境）
2. 低精力养护说明（文字 + 简单图示）
3. 成长时间线 / 应用界面图
4. 价格区间（最后再给）

### 概念介绍话术

```
"假设有一盆桌面植物系统，它不是电子宠物，也不是普通花盆。

它做三件事：

第一，帮植物活下来——顶部补光，底部自动供水。你不用每天判断浇多少，不用搬去晒太阳。

第二，帮你看到变化——记录新叶、花苞、开花的时间线，用照片保存。

第三，不打扰你——平时安静，需要处理时才提醒。

基于你刚才说的经历和需求，你觉得它能解决你的问题吗？"
```

### 正面问题

```
Q1：你觉得它解决了你刚才说的问题吗？哪一个最关键？
→ 听用户指出哪个功能对应了他的痛点。

Q2：最吸引你的是哪一点？
→ 让用户自己排优先级。可能回答：活下来、开花、少操心、记录变化、桌面好看。

Q3：哪一点你最不在意？
→ 听用户说不关键的功能。

Q4：你会担心什么？
→ 听用户的风险识别。可能回答：复杂、伪智能、贵、不好用、还是活不了等。
```

### 侧面问题

```
Q5：你会把它当花盆、当植物、当桌搭，还是当一个电子设备？
→ 判断用户对产品的心智定位。

Q6：如果它没有屏幕表情或语音，你会觉得少了什么吗？
→ 测试"克制智能"的接受度。

Q7：如果只有灯光和 App 记录，你能接受吗？
→ 再次确认用户对形式的期待。

Q8：你会不会担心它变成伪智能、花钱买个智商税？
→ 测试品类信任。听用户说他的顾虑。
```

### 根据用户反应微调

**如果用户说"听起来不错但太复杂"**：

```
→ 强调：不复杂。你只需要做两件事：
  1. 大概 7-10 天加一次水。
  2. 偶尔清洁水箱。
  其他的系统自动处理。
```

**如果用户说"怕是伪智能"**：

```
→ 解释：不是电子宠物那种伪智能。我们没有屏幕表情。
  这是实实在在的：补光让它长好，底灌防止死亡。
  记录是为了让你看清楚它真的在变化。
```

**如果用户说"为什么要花这么多钱，普通花盆 ¥50 就够了"**：

```
→ 反问：但普通花盆让你怎么样了？
  你上次说养不活、养不开花。这套系统的成本是在帮你解决失败的风险。
  我们接下来会更准确地说价格。但先判断，如果它真的能让你的植物稳定开花，
  你会不会觉得值？
```

### 判断标准

| 反应 | 含义 |
|------|------|
| "听起来不错" | 弱吸引力 |
| "这解决了我最烦的问题" + 能具体说出是哪个 | 中等吸引力 |
| "我想试试" 或 "会不会有机会体验一下" | 强吸引力 ✓ |
| "我担心 X，其他都可以" | 有顾虑但可接受 |
| "这就是伪智能" 或 "太复杂了" | 方案需要调整 |

## 九、第 6 层：价格和行动意愿

**目标**：从"喜欢"转到"愿不愿付出成本"。从口头许诺转到实际行为。

**时长**：8-10 分钟

### 不要直接问"你愿意买吗"

### 先问已有支出

```
Q1：你之前桌面绿植/花盆/桌搭单次最高花过多少钱？
→ 听用户说历史支出。了解消费习惯和承受能力。

Q2：你觉得一盆普通桌面绿植多少钱合理？
→ 参考点。如果回答¥30，用户的价格锚点很低。

Q3：你觉得一个带补光和自动浇水的桌面系统应该多少钱？
→ 用户的心理价格。

Q4：如果它能稳定养活一盆会开花的植物（成活率 >90%），你能接受的价格区间是多少？
→ 加上价值支撑后，用户愿意出的价。
```

### 区间测试（不要一次问一个价格）

```
按顺序问：

"¥299 你觉得像什么？"
A 像玩具/普通花盆 → 价格太低，用户觉得廉价。
B 可能可以 → 价格锚点偏低。
C 可以，挺合理的 → 可接受。

"¥499 呢？"
A 有点贵 → 用户抗性上升。
B 可以考虑 → 进入考虑范围。
C 很值 → 用户觉得物有所值。

"¥699 呢？"
A 太贵了，不会买 → 预算上限≤¥699。
B 什么条件下会考虑？ → 还有顾虑，需要条件。
C 可以，如果真的有效果 → 愿意投资。

"¥899 呢？"
A 完全不可能 → 预算天花板≤¥899。
B 作为礼物可能可以 → 个人消费≠礼物消费。
C 长期使用应该值 → 用户认可高价值。
```

### 更好的问法：行为测试

不要再问"你愿意买吗"，改问**具体行为**：

```
Q5：如果现在可以预约 30 天试用（只需留个邮箱），你愿意报名吗？
→ 最低行动成本。看用户是否有意愿。

Q6：如果要交 ¥49 意向金抵扣购买（相当于承诺你有购买意图），你会不会交？
→ 真实支付信号。口头许诺 vs 真金白银。

Q7：如果只开放 20 个测试名额，你想不想参与？（排队等）
→ 稀缺性测试。看用户的迫切度。

Q8：你愿意把它放在你真实桌面上试 30 天吗？（有风险、可能不适合）
→ 最终行动意愿。用户敢不敢让产品进入真实生活。
```

### 判断标准

| 用户反应 | 信号强度 | 含义 |
|---|---|---|
| 只说"好看""有意思" | ⚪ 弱 | 口头兴趣，实际意愿低 |
| 愿意留联系方式/报名试用 | 🟡 中 | 有真实兴趣，但还在观察 |
| 愿意接受明确价格区间（如¥499-699）并说"可以" | 🟠 较强 | 已进入商业考虑 |
| 愿意交意向金 / 排队等名额 | 🔴 强 | 最强行为信号 |
| 只接受很低价（¥199 以下）或无法覆盖成本的价格 | ⚠️ 商业风险 | 价格锚点过低，后续难以维系 |
| 说"只要能活就行""不要太贵"但无具体价格 | ⚪ 弱 | 需求不够明确 |

| 行动意愿 | 只说好 | 愿意试用/报名 | 交了意向金/排队等 | _ |

### 总分计算

```
总分 = 各维度得分之和

0-8 分   → 非目标用户 / 对照用户
9-12 分  → 可观察用户 / 边界用户
13-16 分 → 高匹配用户 ✓
```

### 访谈整体判断

**单个用户判断**：

```
高匹配用户的特征：
□ 固定桌面 ≥4小时/天
□ 过去为桌面/植物付费（¥50+）
□ 有植物失败/妥协经历，且在意
□ 明确想再养会开花的真植物
□ 对"新叶/开花"有明确期待
□ 明确表示不想高频照料
□ 理解方案不是伪智能
□ 能接受 ¥400-800 价格范围
□ 有具体行动意愿（试用、交意向金）
```

如果上述 9 个条件**至少满足 7 个**，判定为"高匹配用户"。

**多个用户汇总判断**：

```
目标用户确认标准（RAT 通过条件）：
□ 连续 5-8 个用户都是"高匹配"
□ 他们对痛点的描述高度相似
□ 他们对成长价值的需求一致
□ 他们对低精力约束的表达相似
□ 至少 3 个用户有具体行动意愿（试用 / 意向金）
□ 没有出现用户完全否定方案或价格
```

如果以上 6 个条件**全部满足**，则判定为"目标用户存在 ✓"，可进入 POC 和样机阶段。

## 访谈问题排查清单

在开始访谈前，自检：

- [ ] 是否准备了开放题？不要一上来问"你喜不喜欢"
- [ ] 是否准备了追问？在用户回答后问"为什么""具体是……"
- [ ] 是否会讲产品？第 5 层之前尽量少讲，多听用户讲
- [ ] 是否有概念图和价格卡？用视觉辅助，不依赖口头解释
- [ ] 是否会区分"口头说"和"真实行为"？要拿出笔记本记录用户的具体行为信号
- [ ] 是否控制时长？45-60 分钟内完成，不要超长访谈导致用户疲劳
- [ ] 是否会记录用户的非语言反应？眼神、语速、停顿都是信号
